# P3-06U-02 角色化任务路径重排

日期：2026-07-01  
适用系统：万法常世 AI 全智能客服系统标准运营版  
上游依据：`P3-06U_FRONTEND_BACKEND_ALIGNMENT_AND_PRACTICAL_UX_OPTIMIZATION_PLAN.md`、`P3-06U-01_FRONTEND_BACKEND_CONTRACT_MATRIX.md`  
阶段定位：把登录后的中台体验从“功能目录”推进到“岗位任务路径”。

## 1. 工程控制卡

| 项目 | 内容 |
| --- | --- |
| 阶段编号 | P3-06U-02 |
| 阶段名称 | 角色化任务路径重排 |
| 本轮目标 | owner/admin/agent/viewer 登录后看到 3-5 条最重要的任务路径，并能直接进入对应工作区 |
| 本轮改动边界 | 不改后端状态机；不打开真实外发；不新增无契约页面；不改变后端 RBAC 安全边界 |

## 2. 角色路径设计

### owner

| 顺序 | 任务路径 | 入口 | 目的 |
| --- | --- | --- | --- |
| 1 | 判断今日风险 | `#overview` | 看进线、人工压力、待发送、异常和健康分 |
| 2 | 处理高风险草稿 | `#reviews` | 先处理会影响回复质量和风险的草稿 |
| 3 | 确认待发送 | `#outbox` | 确认发送门禁和外发关闭边界 |
| 4 | 修复知识缺口 | `#gaps` | 把错因修回知识库 |
| 5 | 检查渠道接入 | `#channels` | 看官方通道、回调验证和失败复盘 |

### admin

| 顺序 | 任务路径 | 入口 | 目的 |
| --- | --- | --- | --- |
| 1 | 判断今日风险 | `#overview` | 一眼判断今日接待压力和异常 |
| 2 | 处理高风险草稿 | `#reviews` | 管理低置信、高风险和无知识命中的回复 |
| 3 | 确认待发送 | `#outbox` | 管理发送前门禁 |
| 4 | 复盘质量错因 | `#quality` | 从质量视角定位知识、渠道和审核问题 |
| 5 | 修复知识缺口 | `#gaps` | 把复盘结果转为知识修复动作 |

### agent

| 顺序 | 任务路径 | 入口 | 目的 |
| --- | --- | --- | --- |
| 1 | 接待客户会话 | `#live` | 坐席先进入真实工作台，而不是先看经营页 |
| 2 | 处理高风险草稿 | `#reviews` | 处理需要人工判断的回复 |
| 3 | 确认待发送 | `#outbox` | 确认已放行草稿进入待发链路 |
| 4 | 跟进客户线索 | `#leads` | 把采购、报价、试点咨询转为可跟进线索 |
| 5 | 跟进工单时效 | `#tickets` | 处理持续售后和 SLA 风险 |

### viewer

| 顺序 | 任务路径 | 入口 | 目的 |
| --- | --- | --- | --- |
| 1 | 判断今日风险 | `#overview` | 只看经营信号，不进入写动作 |
| 2 | 复盘质量错因 | `#quality` | 看 AI 质量、转人工和异常走势 |
| 3 | 检查渠道接入 | `#channels` | 看官方渠道状态和异常，不修改配置 |

## 3. 前端实现

| 文件 | 改动 |
| --- | --- |
| `frontend/src/data/navigation.ts` | 新增 `RoleTaskPath`、`roleTaskPaths`、`getRoleTaskPathsForRoles()`；按角色排序并限制 3-5 条任务路径 |
| `frontend/src/App.tsx` | 新增 `RoleTaskPathStrip`，登录后在顶部工作区展示当前角色任务路径；每条路径链接到现有工作区 |
| `frontend/src/styles.css` | 新增任务路径条样式；桌面保持紧凑横向网格，移动端横向滚动 |
| `scripts/check_p3_06u_02_role_task_paths.py` | 新增静态检查，锁定角色路径、入口顺序、客户化文案和样式存在 |
| `scripts/check_p3_06u_02_role_task_paths.mjs` | 新增浏览器检查，覆盖演示入口、桌面/窄桌面/手机视口、任务路径可见性和横向溢出 |

## 4. 实时数字口径

| 任务路径 | 数字来源 | 说明 |
| --- | --- | --- |
| 判断今日风险 | 服务端运营总览健康分；无聚合时使用当前已加载会话信号 | 不返回客户原文 |
| 处理高风险草稿 | 人工审核 inbox 中 `risk_level=high/critical` 和全部待审数量 | 只作为工作量提示 |
| 确认待发送 | `outbox_drafts` 中待确认与已准备发送数量 | 真实外发仍关闭 |
| 接待客户会话 | 运营聚合入站会话；无聚合时使用审核和待发信号 | 不代表全平台真实接通 |
| 跟进客户线索 | 线索列表总数 | 服务客服转化，不扩展为重 CRM |
| 跟进工单时效 | 工单列表总数 | 保持轻量 SLA |
| 复盘质量错因 | 高风险审核、渠道异常和知识缺口合计 | 后续 P3-06U-06 做更强错因 BI |
| 修复知识缺口 | 知识缺口列表总数 | 指向知识运营闭环 |
| 检查渠道接入 | 失败复盘和阻断 delivery job 数 | 指向渠道连接器中心 |
| 检查运维健康 | 只读运维状态 | 不触发外部动作 |

## 5. 当前边界

- 本轮只是角色任务路径第一片，不代表所有页面信息架构已最终完成。
- 前端路径条不替代后端 RBAC。用户点击无权限页面时仍由现有可见性和后端权限控制。
- viewer 只获得查看路径，不展示审核、待发送、知识修复等写动作主路径。
- agent 默认任务路径第一项是 `#live`，符合坐席日常处理顺序。
- 真实外发继续关闭；任务路径中的“确认待发送”仍是发送前门禁，不等于写入外部平台。
