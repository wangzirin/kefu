# 万法常世 AI 客服本地试点启动说明

## 适用范围

这份说明用于小微企业本地受控试点。它帮助负责人在自己的电脑或局域网服务器上启动客服中台、创建首任负责人账号、完成第一次诊断和备份，再进入知识资料导入。

当前试点包真实外发默认关闭，入站 worker 也默认关闭。未完成官方渠道授权、测试白名单、回执和审计前，系统不会向微信、抖音、淘宝、京东、拼多多等平台自动发送消息。

## 启动前准备

1. 安装并启动 Docker Desktop。
2. 进入系统目录里的 `deploy` 文件夹。
3. 复制 `customer.env.example`，命名为 `customer.env`。
4. 打开 `customer.env`，把 `STANDARD_OPS_POSTGRES_PASSWORD` 和 `DATABASE_URL` 里的模板数据库密码替换成本地随机密码。
5. 不要填写默认管理员密码。首任负责人账号需要在本地登录页创建。

## 启动方式

双击：

```text
deploy/start-local-pilot.command
```

启动完成后打开：

```text
http://127.0.0.1:5173
```

第一次进入时，创建首任负责人账号。后续新增成员、重置密码、停用账号、诊断包、备份、更新和回滚都在“账号与本地维护”里处理。

## 首次启动后的检查

1. 创建首任负责人账号。
2. 登录工作台。
3. 进入“账号与本地维护”。
4. 生成一次本地诊断包。
5. 创建并校验一次本地备份。
6. 进入“知识库运营”，导入试点知识资料包。
7. 进入“知识评测”和“质量复盘”，查看导入后的复测结果。

## 重要边界

- 当前不是完整桌面安装器或正式生产安装包。
- 当前不是正式客户准确率签收。
- 当前不自动连接企业微信、微信客服、抖音、淘宝、京东、拼多多等真实平台。
- 当前不启用真实平台自动外发。
- 当前不预置默认管理员密码。
- 诊断包和备份需要由负责人在本地界面主动生成。

## 遇到启动失败

- 如果提示没有 Docker，请先安装并启动 Docker Desktop。
- 如果提示缺少 `customer.env`，请从 `customer.env.example` 复制。
- 如果提示数据库密码仍是模板值，请替换成本地随机密码。
- 如果端口被占用，请联系交付人员处理，不要自行删除数据库目录。
