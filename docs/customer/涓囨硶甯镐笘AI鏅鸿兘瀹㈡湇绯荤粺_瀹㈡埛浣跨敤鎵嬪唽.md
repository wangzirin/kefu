# 万法常世 AI 全智能客服系统客户使用手册

版本：初版  
适用版本：Lite 试点版、标准运营版  
用途：帮助使用团队完成日常接待、知识库维护、AI 草稿审核、质量复盘和问题反馈。

## 1. 系统能做什么

万法常世 AI 全智能客服系统用于把常见咨询、产品资料、售后规则和人工坐席流程整合到一个可审计的客服工作台中。

核心能力：

- 接收来自官网或授权渠道的咨询。
- 根据已审核知识库生成 AI 回复草稿。
- 展示回复引用的知识来源。
- 对低置信、高风险、售后争议、赔付、合同、法务等问题转人工审核。
- 将审核通过的回复进入待发送区，并保留审计记录。
- 通过题库和质量报告发现知识缺口、错误回答和需要人工处理的问题。

系统不是无人监管的自动回复工具。正式启用自动发送前，应先完成知识库审核、题库回归、人审规则确认和渠道外发授权。

## 2. 角色说明

| 角色 | 主要职责 |
| --- | --- |
| 管理员 | 配置账号、渠道、知识库、模型、质量评测和运维设置 |
| 运营负责人 | 维护知识库、查看质量报告、复盘低置信和知识缺口 |
| 坐席 | 审核 AI 草稿、改写回复、处理高风险和转人工会话 |
| 业务负责人 | 确认产品政策、价格、售后规则和禁用话术 |
| 技术维护人员 | 处理部署、备份、恢复、升级、日志和故障排查 |

## 3. 每日工作流程

### 3.1 登录系统

1. 打开系统地址。
2. 输入企业标识、邮箱和密码。
3. 登录后进入客服工作台。
4. 如果无法登录，先联系管理员确认账号、角色和密码状态。

### 3.2 查看今日待办

进入工作台后，优先查看以下队列：

| 队列 | 应该怎么处理 |
| --- | --- |
| 待人工审核 | 检查 AI 草稿和引用证据，确认能否发送 |
| 待发送确认 | 发送前再次确认内容、渠道和联系人 |
| 发送失败 | 查看失败原因，决定重试、改写或联系技术维护 |
| 知识缺口 | 补充知识库或标记为需要业务负责人确认 |
| 高风险会话 | 投诉、退款、赔付、合同、法务、舆情等必须人工处理 |
| 最近评测运行 | 查看知识库更新后质量是否下降 |
| 渠道健康 | 查看官网、企微、公众号等入口是否正常 |

### 3.3 审核 AI 回复草稿

审核时不要只看草稿文本，必须同时看：

- 用户原始问题。
- AI 草稿。
- 引用证据。
- 风险等级。
- 转人工原因。
- 知识来源。
- 历史会话。
- 待发送状态和审计记录。

可以直接批准的情况：

- 问题属于低风险常见咨询。
- 知识库证据明确。
- 草稿没有承诺最低价、赔付、法律判断、医疗建议等高风险内容。
- 语气符合企业服务口径。

必须改写或转人工的情况：

- 草稿没有引用证据。
- 草稿与现行政策不一致。
- 用户要求退款、赔偿、合同解释、投诉处理或法务承诺。
- 用户提供了个人隐私、订单争议或敏感信息。
- 问题超出知识库范围。

### 3.4 处理待发送

审核通过不等于已经发送。待发送区用于最终确认：

1. 检查收件渠道。
2. 检查联系人。
3. 检查最终回复内容。
4. 确认是否允许该渠道发送。
5. 再执行发送或模拟发送。

如果系统显示外发关闭，代表当前只做内部审核和演示，不会向真实平台发送消息。

## 4. 知识库使用方法

### 4.1 知识库包含什么

知识库通常包括：

- 产品介绍。
- 价格和套餐。
- 交付周期。
- 售后政策。
- 退款和保修规则。
- 发票和合同说明。
- 账号权限。
- 渠道接入说明。
- 禁用话术和高风险边界。

### 4.2 如何新增知识

推荐流程：

1. 业务负责人提供资料。
2. 运营负责人整理为知识卡片或文档。
3. 管理员导入系统。
4. 系统自动生成检索片段和引用。
5. 运行题库回归。
6. 人工抽查 AI 草稿。
7. 确认无误后发布。

### 4.3 如何更新知识

价格、活动、售后政策、合同条款发生变化时，应按版本更新：

- 写明变更原因。
- 写明生效时间。
- 保留旧版本。
- 更新相关题库。
- 跑一次质量回归。
- 通知坐席新口径。

不要直接覆盖旧知识。没有版本记录，后续很难追溯错误回复来自哪个政策版本。

## 5. 质量复盘方法

系统质量不应只看“回答了多少”，而要看“回答是否有证据、是否需要人工、是否被坐席大幅改写”。

建议每周查看：

| 指标 | 含义 |
| --- | --- |
| 知识命中率 | 用户问题能否找到相关知识 |
| 引用支撑率 | AI 草稿是否被知识库证据支持 |
| 转人工率 | 有多少问题需要人工处理 |
| 低置信率 | 系统不确定的问题比例 |
| 坐席改写率 | AI 草稿被人工大改的比例 |
| 知识缺口数 | 知识库缺失的问题数量 |
| 发送失败数 | 渠道或权限导致的发送异常 |
| 负反馈数 | 用户、坐席或业务负责人标记的问题 |

出现准确率下降时，按以下顺序排查：

1. 最近是否更新了产品、价格、售后政策。
2. 最近是否更新了知识库。
3. 最近是否更换了模型或模型参数。
4. 最近是否新增了渠道或用户来源。
5. 最近是否有大量新类型问题。
6. 题库回归结果是否下降。
7. 坐席改写率是否升高。

## 6. 常见问题

### AI 会不会自己乱发消息

系统默认以人工审核和外发门禁为安全边界。未经配置和授权，AI 草稿不会直接发送到真实渠道。

### 知识库更新后多久生效

取决于部署方式和索引策略。一般轻量知识卡片可较快生效；文档知识更新建议先跑评测，再正式发布。

### 为什么有些问题被转人工

常见原因包括低置信、无知识命中、涉及退款赔付、合同法务、投诉舆情、隐私信息或超出知识库范围。

### 能否接企业微信、公众号、抖音、小红书、淘宝等平台

可以逐步接入，但必须走官方 API、开放平台或服务商授权。个人号外挂、Hook、群控和模拟点击不属于正式交付方案。

### 系统上线后谁负责维护

系统维护通常由使用团队和服务方共同完成：

- 使用团队负责提供真实业务知识、确认政策口径、审核高风险内容。
- 服务方负责系统部署、技术维护、质量复盘支持和故障处理。

## 8. 试点期建议

前 2 周建议保持：

- 不开放全自动外发。
- 所有 AI 草稿先人工审核。
- 每日复盘知识缺口。
- 每周输出质量报告。
- 先稳定一个入口，再扩展更多渠道。

试点稳定后，再根据题库回归、人工审核结果和业务风险决定是否开放低风险自动回复。
