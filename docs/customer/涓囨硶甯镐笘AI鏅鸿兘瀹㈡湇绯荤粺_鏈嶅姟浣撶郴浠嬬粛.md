# 万法常世 AI 全智能客服系统服务体系介绍

版本：初版  
适用对象：企业负责人、项目负责人、客服负责人、运营负责人

## 1. 服务目标

智能客服系统不是一次性安装软件。上线后的效果取决于知识库质量、业务口径更新、模型质量、渠道稳定性和人工复盘。

服务体系的目标是：

- 帮助企业完成从部署到试运营的落地。
- 帮助客服团队掌握 AI 草稿审核和知识库维护。
- 帮助管理者持续看到质量指标和问题闭环。
- 帮助系统稳定运行，出现问题时可诊断、可恢复、可回滚。

## 2. 服务内容总览

| 服务模块 | 内容 |
| --- | --- |
| 需求梳理 | 业务范围、入口渠道、知识资料、风险边界、试点目标 |
| 系统部署 | 后端、前端、数据库、队列、模型配置、基础账号 |
| 知识库建设 | 资料整理、知识卡片、文档导入、禁用话术、版本口径 |
| 题库建设 | 50-100 条脱敏问题、期望证据、人审边界、风险标签 |
| 坐席培训 | 审核 AI 草稿、改写回复、处理高风险、记录知识缺口 |
| 质量复盘 | 命中率、引用支撑率、转人工率、知识缺口、负反馈 |
| 运维维护 | 健康检查、日志、备份、恢复、升级、回滚、诊断包 |
| 渠道支持 | 官网入口和官方授权渠道的接入、回调、验签、故障排查 |

## 3. 实施服务

### 3.1 启动阶段

需要确认：

- 试点业务范围。
- 入口渠道。
- 使用人员名单。
- 知识资料来源。
- 不允许 AI 自动回答的问题。
- 是否允许调用外部模型。
- 是否开启真实外发。

输出：

- 项目启动清单。
- 试点范围确认。
- 资料收集清单。
- 风险边界确认。

### 3.2 部署阶段

服务内容：

- 部署应用服务。
- 配置数据库。
- 配置模型服务。
- 配置基础账号。
- 配置官网或授权渠道入口。
- 设置外发开关。
- 配置备份和日志。

输出：

- 部署记录。
- 环境变量清单。
- 账号初始化记录。
- 健康检查结果。
- 回滚点。

### 3.3 知识库建设阶段

服务内容：

- 整理产品、价格、售后、交付、账号、合同、发票等资料。
- 转换为知识卡片或知识文档。
- 标记禁用话术和高风险边界。
- 导入系统。
- 运行题库回归。

输出：

- 知识库初版。
- 知识来源清单。
- 题库回归报告。
- 知识缺口清单。

### 3.4 试运营阶段

建议试运营前 2 周保持人工审核，不默认自动外发。

服务内容：

- 观察真实咨询。
- 记录坐席改写。
- 收集知识缺口。
- 分析低置信问题。
- 修订知识库。
- 复盘模型草稿质量。

输出：

- 试运营周报。
- 知识库修订清单。
- AI 草稿质量记录。
- 下一阶段优化建议。

## 4. 运维服务

### 4.1 日常运维

| 项目 | 内容 |
| --- | --- |
| 健康检查 | 服务、数据库、队列、模型服务、渠道回调 |
| 日志检查 | 应用日志、审计事件、发送尝试、失败复盘 |
| 备份检查 | 数据库备份、知识库备份、配置备份 |
| 成本检查 | 模型调用量、失败率、延迟、估算费用 |
| 质量检查 | 题库回归、知识缺口、人工审核率 |

### 4.2 故障响应

| 等级 | 示例 | 响应目标 |
| --- | --- | --- |
| P1 紧急 | 系统不可用、数据恢复、错误外发风险 | 优先响应，先止血再复盘 |
| P2 高 | 渠道大面积失败、模型不可用、知识库发布错误 | 当日处理 |
| P3 中 | 单个功能异常、部分坐席无法使用 | 按维护窗口处理 |
| P4 低 | 文案、配置、普通咨询 | 排期处理 |

所有故障处理都应记录：

- 发生时间。
- 影响范围。
- 临时处理。
- 根因分析。
- 修复动作。
- 是否需要知识库或流程更新。

## 5. 质量运营服务

### 5.1 月度质量复盘

建议每月输出一次质量复盘，包含：

- 总咨询量。
- AI 草稿数量。
- 人工审核数量。
- 转人工率。
- 引用支撑率。
- 坐席改写率。
- 知识缺口 Top 问题。
- 高风险问题处理情况。
- 渠道失败情况。
- 下月优化计划。

### 5.2 准确率下降处理

如果出现质量下降，按以下路径排查：

1. 检查最近知识库是否变更。
2. 检查题库回归是否下降。
3. 检查模型服务是否变更。
4. 检查渠道来源是否变化。
5. 检查坐席改写率是否升高。
6. 检查是否出现新业务或新活动问题。
7. 修订知识库和提示词。
8. 重新跑回归。

## 6. 责任边界

服务方负责：

- 系统部署。
- 技术维护。
- 知识库导入支持。
- 质量报告支持。
- 故障排查。
- 升级和回滚方案。

使用团队负责：

- 提供真实业务资料。
- 确认产品、价格、售后和合同口径。
- 审核高风险回复。
- 提供脱敏题库。
- 授权渠道和模型调用。
- 确认是否开启真实外发。

双方共同负责：

- 试点目标。
- 风险边界。
- 知识更新周期。
- 自动回复开放范围。
- 月度复盘和优化计划。

## 7. 服务分层建议

| 服务层级 | 适合版本 | 内容 |
| --- | --- | --- |
| 基础维护 | Lite 试点版 | 健康检查、基础故障、月度质量摘要、知识小修 |
| 标准运营 | 标准运营版 | 周/月质量复盘、知识更新、渠道排障、模型策略优化 |
| 专项支持 | 标准运营版及以上 | 私有化部署、多渠道接入、高并发、合规审计、专属 SLA |

## 8. 上线后节奏

建议节奏：

- 第 1 周：部署、知识导入、坐席培训、题库回归。
- 第 2 周：小范围试运营，所有 AI 草稿人工审核。
- 第 3-4 周：复盘真实问题，补知识库，优化提示词和转人工边界。
- 第 2 个月：按质量指标决定是否开放低风险自动回复。
- 第 3 个月：评估是否扩展渠道、坐席和标准运营能力。
