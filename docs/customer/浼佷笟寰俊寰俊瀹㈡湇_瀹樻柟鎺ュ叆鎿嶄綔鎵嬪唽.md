# 企业微信微信客服官方接入操作手册

从创建客服账号、绑定接待人员到配置 URL / Token / EncodingAESKey 的后台操作步骤

- 文档类型：渠道接入操作手册
- 适用对象：项目负责人、企业微信管理员、交付工程师、测试人员
- 版本：v1.0
- 更新时间：2026-06-30

> 核心结论：不要注册个人号机器人；本阶段按企业微信官方微信客服路径，创建客服账号、绑定接待人员、配置可调用接口应用和回调 URL。

## 一、先把概念讲清楚

这一步不是注册个人微信机器人，也不是用外挂托管微信号。我们要走的是企业微信/微信客服官方路径：先创建一个微信客服账号，再把一个可调用接口的自建应用绑定到微信客服，最后把企业微信平台的消息回调接到我们的客服中台。

| 你看到的名称 | 实际含义 | 本阶段怎么处理 |
| --- | --- | --- |
| 微信客服账号 | 客户在微信里发起咨询时看到的客服入口 | 必须创建，并绑定接待人员 |
| 接待人员 | 企业微信成员，负责人工接待或兜底 | 先绑定你自己的企业微信成员 |
| 客服链接/二维码 | 客户发起咨询的入口 | 保存后获取，用个人微信测试 |
| 可调用接口的应用 | 允许某个自建应用调用微信客服 API | 必须设置，选择测试自建应用 |
| 接收消息/回调配置 | 企业微信把事件推送到我们的公网 URL | 填 URL、Token、EncodingAESKey |
| 智能机器人 | 企业微信官方也有机器人相关能力 | 本阶段不依赖它；我们的 AI 在自研客服中台里完成 |

> **当前建议**：你现在先不要找“注册机器人”按钮。先完成微信客服账号、接待人员、客服链接、可调用接口应用、回调 URL 验证。机器人式自动回复在我们系统里完成，不靠个人号或群控。

## 二、当前已准备好的我方回调信息

当前本地测试使用的是临时公网隧道，适合这次 URL 验证和入站 sandbox。隧道关闭、终端停止或地址变化后，需要重新生成 URL。

| 字段 | 当前值或来源 | 说明 |
| --- | --- | --- |
| URL | https://tasty-otters-start.loca.lt/api/webhooks/wecom/channels/1 | 填到企业微信后台的回调 URL；只适合本次本地测试 |
| Token | 见本机临时 env 文件里的 WECOM_KF_CALLBACK_TOKEN | 敏感值，不写进聊天和文档正文 |
| EncodingAESKey | 见本机临时 env 文件里的 WECOM_KF_ENCODING_AES_KEY | 敏感值，不写进聊天和文档正文 |
| 外发状态 | 关闭 | 本阶段只接收消息、生成内部会话和 AI 草稿，不真实发回微信 |

## 三、后台操作总流程

1. 进入企业微信管理后台，打开“应用管理”。
2. 进入“微信客服”。如果还没启用，先启用微信客服能力。
3. 创建或编辑客服账号，例如“万法常世 AI 客服测试”。
4. 设置头像和名称，保存客服账号。
5. 绑定接待人员，先选择你自己的企业微信成员。
6. 保存后回到客服账号列表，获取客服链接或二维码。
7. 回到微信客服应用详情，找到“可调用接口的应用”，点击“设置”。
8. 选择已有自建应用，或新建一个测试自建应用。
9. 进入该自建应用的详情页，找到接收消息、回调配置、API 接收消息或类似入口。
10. 填入 URL、Token、EncodingAESKey，提交保存，让企业微信验证 URL。
11. 验证通过后，用个人微信打开客服链接，发一条测试消息。
12. 回到我们的客服中台或后端日志，确认消息已进入系统。

## 四、详细操作步骤

### 步骤 1：创建或保存微信客服账号

如果你当前页面类似“应用管理 -> 微信客服 -> 应用详情”，并且下面能看到“客服账号”“创建账号”或某个客服账号行，这个页面是对的。先把客服账号本身保存好。

1. 在“客服账号”区域点击“创建账号”，或点击已有客服账号进入编辑。
2. 客服名称填“万法常世 AI 客服测试”或客户项目名。
3. 头像可以先用默认头像，正式试点再换公司标识。
4. 可见范围先保留当前企业或测试部门。
5. 保存客服账号。

### 步骤 2：绑定接待人员

接待人员通常在客服账号编辑页或客服账号列表的详情里。不同后台版本按钮可能叫“接待人员”“添加接待人员”“修改接待人员”“接待设置”。

1. 进入刚创建的客服账号详情。
2. 找到“接待人员”区域。
3. 点击“添加”或“修改”。
4. 选择你自己的企业微信成员。
5. 保存。

> **找不到接待人员时**：先确认你点进的是具体客服账号，而不是微信客服应用总览页；如果页面底部有客服账号列表，点击账号行右侧箭头进入账号详情。

### 步骤 3：获取客服链接或二维码

1. 回到客服账号列表。
2. 点击该客服账号行，或点击右侧“接入场景”“客服链接”“二维码”“在微信内接入”等按钮。
3. 复制客服链接或下载二维码。
4. 先不要发给真实客户，只用自己的个人微信测试。

### 步骤 4：设置可调用接口的应用

企业微信开发者文档明确提到，自建应用需要配置到“微信客服-可调用接口的应用”中，才能调用微信客服相关能力。你截图里已经在“微信客服”应用详情页看到“可调用接口的应用：设置”，这里就是关键入口。

1. 回到“应用管理 -> 微信客服”的应用详情页。
2. 找到“可调用接口的应用”。
3. 点击“设置”。
4. 选择一个测试自建应用；如果没有，就创建一个新的自建应用，名称可用“万法常世 AI 客服测试应用”。
5. 保存后，记住这个自建应用，因为下一步 URL/Token/AESKey 通常在这个应用的接收消息配置里设置。

### 步骤 5：进入自建应用接收消息配置

不同企业微信后台版本入口名称不完全一致，你可以按下面顺序找。

- 路径 A：应用管理 -> 自建应用 -> 选择刚才绑定的应用 -> 接收消息 -> 设置 API 接收。
- 路径 B：应用管理 -> 微信客服 -> 可调用接口的应用 -> 设置 -> 进入应用详情 -> 接收消息。
- 路径 C：应用管理 -> 自建应用 -> 开发者接口 / API / 回调配置 / 接收消息。

### 步骤 6：填写 URL、Token、EncodingAESKey

| 字段 | 怎么填 | 注意事项 |
| --- | --- | --- |
| URL | 填我方给你的公网 HTTPS 回调地址 | 必须是 HTTPS；不能填 127.0.0.1；本轮填 localtunnel 地址 |
| Token | 填本机临时 env 文件中的 WECOM_KF_CALLBACK_TOKEN | 要和后端环境变量完全一致 |
| EncodingAESKey | 填本机临时 env 文件中的 WECOM_KF_ENCODING_AES_KEY | 长度通常为 43 位；要和后端完全一致 |

> **加解密方式**：如果后台提供消息加解密方式选择，优先使用安全模式或兼容安全模式。本系统已经支持 AES 解密，不要把后台模式改成与后端实现不一致的模式。

### 步骤 7：提交验证

1. 确认本地后端还在运行。
2. 确认公网隧道还在运行。
3. 在企业微信后台点击“保存”“提交”或“验证”。
4. 如果验证通过，说明企业微信已经能访问我们的 URL，并且后端能解密返回 echostr。
5. 如果验证失败，先不要反复乱改字段，按本文第六部分排查。

### 步骤 8：发送第一条真实测试消息

1. 用你的个人微信打开客服链接或扫描二维码。
2. 发送一条简单测试消息，例如“你好，我想了解试点部署”。
3. 观察企业微信后台客服会话是否出现。
4. 观察我方后端是否收到消息回调。
5. 确认系统内是否创建会话和消息。
6. 本阶段只生成内部草稿和人工审核，不自动发回微信。

## 五、常见问题和排查

| 问题 | 可能原因 | 处理方式 |
| --- | --- | --- |
| 找不到 URL/Token/AESKey | 还停留在微信客服应用详情，未进入自建应用接收消息配置 | 先点“可调用接口的应用：设置”，进入绑定的自建应用详情 |
| 提示没有可信 IP | 可信 IP 通常影响自建应用调用 API 的出站能力 | 本阶段入站 URL 验证可先走公网隧道；若后台强制要求可信 IP，用云服务器固定 IP |
| URL 验证失败 | 隧道断开、后端停止、Token/AESKey 不一致、URL 路径错误 | 先访问 /health，再检查三项字段是否和后端一致 |
| 个人微信发消息没进系统 | 客服账号未保存、未绑定接待人员、链接不是当前账号、回调配置未生效 | 重新确认客服账号、接待人员、客服链接和回调验证状态 |
| 担心封号 | 使用个人号外挂、Hook、群控、模拟点击会有风险 | 只走企业微信/微信客服官方 API，不做个人号托管 |

## 六、本阶段验收标准

- 企业微信后台 URL 验证通过。
- 个人微信打开客服链接后能发送测试消息。
- 我方系统收到企业微信加密 XML 消息。
- 后端验签、AES 解密、XML 解析成功。
- 系统生成内部会话和消息记录。
- 后续 AI 草稿进入人工审核。
- 真实外发仍保持关闭。
