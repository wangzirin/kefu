## 1. Lite 试点版一句话定义

Lite 试点版是一个“受控 Copilot 客服试点包”：它围绕一个官网或自有入口，把客户问题转成可信入站消息，基于知识库和模型路由生成带引用证据的 AI 草稿，再由人工审核、改写或确认，最后进入 outbox 门禁和审计链。

它的核心价值是验证三件事：

1. 客户常见问题能不能被知识库检索到。
2. AI 草稿能不能显著减少坐席重复写回复的时间。
3. 低置信、高风险、知识不足和外发动作能不能被清晰拦住。

它不是全平台自动回复系统，也不是无人值守客服中心。

## 3. 当前不能承诺

| 不能承诺 | 原因 |
| --- | --- |
| 全平台已经接通 | 当前只完成官网/自有入口 sandbox；企业微信、公众号、电商平台需要官方账号和授权 |
| 默认自动真实外发 | 真实 sender 未实现，`OUTBOX_EXTERNAL_WRITE_ENABLED=false` 必须继续作为默认 |
| 无需人工客服 | 高风险、低置信、无知识支撑、政策不明确问题仍需要人工审核 |
| 高并发 SLA | 当前缺生产级 worker、监控告警、压测和高可用部署 |
| 私有大模型部署 | Lite 默认使用云端模型 API 或 deterministic fallback，不包含本地大模型集群 |
| 100% 准确率 | 知识过期、客户问法变化、模型/provider 波动都会导致质量变化 |
| 真实客户题库验收已完成 | 现有题库是客户式 rehearsal，不是客户真实脱敏历史问题 |
| 复杂工单/SLA/绩效 | 这些属于标准运营版 P3-06 以后的范围 |

## 6. 标准演示流程

### 6.1 售前演示流程

1. 打开登录页，使用试点管理员或坐席账号进入工作台。
2. 展示行动队列：待审核、待发送、发送失败、知识缺口、高风险、最近评测、渠道健康。
3. 展示一条官网/自有入口入站消息。
4. 展示 AI 草稿、知识引用、风险等级和转人工原因。
5. 展示坐席如何改写或确认草稿。
6. 展示 outbox 门禁，说明真实外发默认关闭。
7. 展示知识库导入和文档检索引用。
8. 展示题库回归报告和知识缺口。
9. 展示诊断包和服务体系，说明试点后如何维护。

### 6.2 交付验收流程

1. 确认客户试点入口、域名、管理员、坐席和知识负责人。
2. 导入客户确认过的 Lite 知识包。
3. 跑 50 条题库 dry-run 或小样本回归。
4. 配置模型 key 或保留 deterministic fallback。
5. 完成 10 条真实式问题演示。
6. 确认低置信、高风险、缺知识问题进入人工。
7. 确认 outbox 外发关闭。
8. 生成诊断包。
9. 培训管理员、坐席和知识负责人。
10. 输出试点上线记录和剩余问题清单。

## 8. Lite 与标准运营版边界

| 能力 | Lite 试点版 | 标准运营版 |
| --- | --- | --- |
| 入口数量 | 一个官网或自有入口优先 | 官网 + 一个官方平台 sandbox 起步 |
| 客户数量 | 单客户单实例 | 单客户单实例优先，多租户另立项 |
| 坐席协同 | 基础审核和待发送 | 坐席分配、团队、SLA、绩效逐步补齐 |
| 知识库 | FAQ + 轻量文档 + 基础评测 | 知识版本、审核、回滚、缺口闭环 |
| 模型 | 百炼/千问为主，低置信人审 | 模型路由、fallback、成本、熔断、月报 |
| 渠道 | 自有入口为主 | 官方授权渠道逐步接入 |
| 外发 | 默认不真实外发 | 按白名单、风险等级和授权逐步开放 |
| 运维 | 诊断包、部署清单、月度建议 | 监控、告警、worker、SLA、月报 |
