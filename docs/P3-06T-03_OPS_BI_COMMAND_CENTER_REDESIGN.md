## 本轮修复的问题

P3-06T-02 已经把首页数据口径收紧，但首页视觉仍更像“卡片列表”。本轮重做聚焦三件事：

- 把首页首屏改为经营指挥舱，而不是功能卡片堆叠。
- 让运营人员一眼看出今天的入站压力、AI 覆盖、人工压力、异常和知识缺口。
- 让下一步动作可点击可定位：高风险审核、待发送草稿、知识和渠道复盘。

## 取舍说明

- 1280 和 1440 宽屏要求首屏能看到完整经营信号和主指挥舱主体。
- 900 宽度因为左侧中台导航仍占用空间，右侧实际工作区只有六百多像素，不强行要求完整主指挥舱全部塞进第一屏；验收标准改为经营信号和指挥舱入口可见，完整内容在右侧 `.workspace` 独立滚动中查看。
- 手机端不强塞桌面 BI 结构，保留自然滚动和横向紧凑导航。

## 剩余风险

- 本轮仍是前端 BI 重做，不是生产历史数仓或统计缓存。
- 运营总览图表配置仍集中在 `App.tsx`，后续应拆到 `components/ops` 和 `components/charts`。
- 总览页仍有内部演示态提示；正式客户环境需要租户账号登录后进入真实聚合口径。
- 信息架构收口已并入 P3-06U；下一步优先用前后端契约矩阵继续校准导航、角色路径和页面任务流，避免导航和页面继续有工程阶段感。
