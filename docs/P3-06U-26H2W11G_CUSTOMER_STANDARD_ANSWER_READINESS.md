# H2W-11G 客户标准答案口径准备

## 定位

H2W-11G 接在 H2W-11F 之后，只补“客户标准答案口径”的模板和门禁。

上一阶段已经把知识维护路径收束成客户能理解的“整理资料、检查、导入、启用、复测、确认”。这一阶段不继续改页面，也不接真实渠道，而是先把“什么样的回答算正确”做成可追溯模板。后续拿到客户脱敏资料时，质量评测不能只看必含词、检索命中或页面报告，而要能对照客户确认过的标准答案口径、禁用承诺、转人工规则和引用来源。

1. 新增客户标准答案模板：
   - 文件：`evals/p3_06u_26h2w11g_customer_standard_answer_template.csv`
   - 模板包含：场景、问题摘要、业务对象、标准答案、必含词、禁用词、引用来源、是否允许自动回复、是否应转人工、负责人、复核日期、样本来源和客户确认状态。

2. 新增 H2W-11G 门禁：
   - 文件：`scripts/check_p3_06u_26h2w11g_customer_standard_answer_readiness.py`
   - 校验列顺序、样本数、标准答案长度、必含词、引用来源、转人工原因、禁用词回避、样本来源、客户确认状态和常见敏感信息形态。

3. 新增脚本级测试：
   - 文件：`backend/tests/test_p3_06u_26h2w11g_customer_standard_answer_readiness.py`
   - 防止这一步被误写成正式客户准确率签收。

4. 继续读取上游事实：
   - H2W-11F 必须 `passed`。

   - H2W-11B 的 50 条以上受控演练样本仍必须存在。

## 真实口径

本片准备的是客户标准答案口径，不是正式客户准确率签收。

当前模板仍是合成/脱敏结构样例，用来固定后续资料格式、验收规则和禁止承诺边界。真实客户原始数据仍未使用。正式试点前仍需要客户确认样本来源，或提供脱敏真实问题和对应标准答案，再跑最终客服答案质量评测。

最终客服答案质量至少要看四件事：

- 事实正确：回答是否符合客户确认的标准答案。
- 引用充分：回答是否能追到知识卡、文档、版本或来源 URI。
- 禁用承诺：是否避开赔付、效果、渠道风险、平台规则等禁止承诺。
- 转人工正确：高风险、资料不足或无引用时是否进入人工处理。

## 停止门禁

出现任一情况，本片不能通过：

- 标准答案模板缺少必需列或列顺序错位。
- 样本少于 10 条，无法覆盖基础售前、渠道、售后、知识维护、成本、引用和验收边界。
- 标准答案直接复述禁用承诺词。
- 应转人工的问题仍标记允许自动回复。
- 要求引用的问题没有来源 URI。
- 模板出现手机号、邮箱、身份证号等疑似敏感信息。
- 样本来源被写成真实客户数据，但没有客户确认。
- 上游 H2W-11F 未通过。
- 上游 H2W-11B 不是 `controlled_trial_ready`。
- 页面或文档把本片写成完整线上准确率、正式签收、正式电子签章、生产上线或真实外发已完成。

## 本片不做

- 不导入真实客户原始数据。
- 不打开真实外发。
- 不接企业微信、公众号、抖音、淘宝、京东、拼多多真实渠道。
- 不调用真实模型供应商。
- 不生成正式客户签收、合同签收或电子签章。
- 不宣称完整线上准确率。
- 不新增前端按钮或后端业务接口。
