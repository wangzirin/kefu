# P3-06U-11A 本地真实开发库与刷新稳定化

## 目标

把“真实本地账号、真实租户、真实会话样本、人工审核、待发送、知识缺口、工单和线索”固定成可重复运行的开发底座，避免前端评审继续停留在空态或纯演示态。

  - `scripts/seed_p3_06u_11_local_dev_workspace.py`
- 固定本地租户：
  - `wanfa-local-dev`
- 固定本地测试账号：

  - 4 条会话

  - 2 条待发送草稿
  - 1 条发送失败复盘
  - 2 条知识缺口
  - 1 条工单
  - 1 条线索
- 对话台新增“刷新队列”动作。
- 进入 `多渠道对话台`、`会话收件箱`、`人工审核`、`待发送草稿` 时自动刷新会话、人审、待发送、发送失败和发送队列数据。

## 凭据边界

脚本不会打印密码或 token。

如需让测试账号具备可手动输入的密码，运行时显式传入：

```bash
WANFA_LOCAL_DEV_PASSWORD='本地临时密码' backend/.venv/bin/python scripts/seed_p3_06u_11_local_dev_workspace.py
```

运行后脚本只会说明密码来源为 `env_password_hash`，不会输出明文密码。

如果没有传入 `WANFA_LOCAL_DEV_PASSWORD`，且账号不存在，脚本会生成一个随机密码并只保存哈希。此时账号是真实账号，但不能靠人工输入密码登录；需要重新运行脚本并传入本地临时密码。

## 使用方式

推荐在项目根目录运行：

```bash
backend/.venv/bin/python scripts/seed_p3_06u_11_local_dev_workspace.py
```

当前全局 `python3` 未必安装后端依赖，必须优先使用项目后端虚拟环境。

## 验收

运行：

```bash
python3 scripts/check_p3_06u_11a_local_dev_workspace.py
```

检查项包括：

- 种子脚本存在且不打印密码或 token。
- 固定租户 `wanfa-local-dev` 存在。
- 本地库至少有账号、角色、渠道、联系人、会话、人审、待发送、知识缺口、工单和线索。
- 前端 `多渠道对话台` 有刷新入口。
- 切换到会话相关页面时会刷新真实服务端数据。

## 产品判断

这一步不是让系统直接具备生产商用能力，而是让后续 UI、信息架构和前后端对齐评审有真实本地数据可看。没有真实样本，很多页面会因为空态而显得“还可以”；一旦放入真实会话，入口重复和坐席路径割裂才会暴露出来。
