# P3-06U-11C 微信式多渠道对话台第一片

## 目标

让 `多渠道对话台` 更接近真实客服坐席日常使用的聊天窗口，而不是管理看板。普通坐席进入后应先看到客户列表和对话区，不应先被任务路径、运行说明、状态条和大量管理信息占据。

## 已完成

- `多渠道对话台` 不再展示顶部“今日任务路径”。
- `多渠道对话台` 不再展示全局运行状态条。
- 演示提示不再压在对话台顶部，数据来源和真实外发边界由对话台内部的轻量标识表达。
- 顶部全局栏在对话台里压缩：
  - 隐藏副标题说明。
  - 隐藏连接卡大块信息。
  - 保留检查连接和退出。
- 对话台自身标题压缩：
  - 隐藏“坐席接待”小标签。
  - 标题说明收起。
  - 摘要数字改成轻量小标签。
  - 增加“刷新队列”按钮。
- 对话区上下文卡片从卡片式信息块压缩为轻量上下文条。
- 桌面端对话台可用高度从 `calc(100vh - 220px)` 提升到 `calc(100vh - 154px)`，让客户列表和聊天区更接近一屏工作台。

## 暂不删除的能力

本轮没有删除以下路由：

- 会话收件箱
- 人工审核
- 待发送草稿
- 工单/SLA

原因是这些页面仍然对应后端状态机、安全门禁或标准运营版管理动作。下一步应做的是“导航降级和职责重分配”，不是直接删掉。

## 产品判断

坐席主界面应是：

```text
左侧：客户列表、渠道、未读/待审/超时状态
右侧：当前客户对话、AI 草稿、人工备注、批准进入待发送
```

质量分析、知识缺口、渠道授权、运维告警和模型路由不应挤在聊天窗口里。它们属于独立运营页面。

## 下一步

进入 P3-06U-11B 或 P3-06U-11D：

- P3-06U-11B：把工作台左侧导航降级，普通坐席默认只看到 `多渠道对话台`，管理员再看到审核、待发送和工单入口。
- P3-06U-11D：收束运营总览底部“会话处置详情”和“渠道健康”，改成摘要 + 下钻。
