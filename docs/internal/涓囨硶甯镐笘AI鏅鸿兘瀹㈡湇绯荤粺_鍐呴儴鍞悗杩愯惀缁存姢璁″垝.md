# 万法常世 AI 全智能客服系统内部售后运营维护计划

版本：初版  
使用对象：我方销售、交付、工程、运维、客户成功  
注意：本文为内部文档，不对客户直接发送。

## 1. 内部目标

卖出 Lite 试点版或标准运营版后，不能只靠临时救火维护。内部必须形成稳定流程：

- 销售承诺可追溯。
- 交付范围可验收。
- 客户资料可脱敏。
- 远程维护可授权、可审计、可撤销。
- 知识库更新可回归、可回滚。
- 准确率下降可侦测、可定位、可修复。
- 运维成本可估算、可报价、可复盘。

## 2. 销售到交付交接

销售签单或进入试点前，必须交接：

| 项目 | 必填 |
| --- | --- |
| 客户名称 | 是 |
| 行业和业务范围 | 是 |
| 购买版本 | Lite 试点版 / 标准运营版 |
| 承诺入口 | 官网 / 企微 / 公众号 / 其他 |
| 是否承诺真实外发 | 必须明确，默认否 |
| 是否承诺官方平台接入 | 必须明确，需客户授权 |
| 是否涉及真实客户数据 | 必须明确，需脱敏 |
| 是否允许外部模型调用 | 必须明确 |
| 预计咨询量 | 是 |
| 预计坐席数 | 是 |
| 维护服务等级 | 基础 / 标准 / 专项 |
| 特殊合规要求 | 是 |

禁止销售口径：

- 全平台都能马上自动回复。
- 不需要人工客服。
- 准确率 100%。
- 个人微信也能稳定接。
- 不需要客户整理知识库。
- 不需要后续维护成本。

## 3. 交付启动清单

交付前必须收集：

- 企业基础信息。
- 管理员账号信息。
- 坐席名单。
- 业务资料。
- 产品和价格表。
- 售后政策。
- 合同和发票口径。
- 禁用话术。
- 高风险问题清单。
- 50-100 条真实脱敏问题。
- 渠道授权材料。
- 模型 key 或模型采购方式。

不得收集或写入长期文档：

- 明文密码。
- API key。
- token。
- 私钥。
- webhook secret。
- 客户未脱敏聊天记录。
- 身份证、手机号、地址等个人敏感信息原文。

## 4. 远程维护方案

优先级从高到低：

1. 客户托管在我们云端，由我们负责运维。
2. 客户私有化但提供 VPN、零信任或堡垒机临时授权。
3. 客户私有化并按次提供临时远程桌面。
4. 只导出诊断包，由我们离线分析。

不推荐：

- 长期开公网 SSH。
- 长期开公网数据库端口。
- 让我们长期持有客户生产 root 账号。
- 通过个人聊天软件发送生产密钥。

远程维护必须记录：

- 授权人。
- 授权时间。
- 授权范围。
- 操作人。
- 操作内容。
- 是否涉及生产数据。
- 是否有回滚点。
- 处理结果。

详细故障分级、授权单、只读命令白名单、禁止命令、变更二次授权和故障复盘模板，以 `REMOTE_MAINTENANCE_AUTHORIZATION_SOP.md` 为准。

## 5. 诊断包标准

诊断包应包含：

- 系统版本。
- 部署模式。
- 环境变量名称和值状态，不包含敏感值。
- 健康检查结果。
- 最近错误摘要。
- 队列状态。
- 模型 provider 状态。
- 渠道连接器状态。
- 最近 50 条审计事件摘要。
- 最近发送失败摘要。
- 知识库版本摘要。
- 最近一次质量评测摘要。

诊断包不应包含：

- API key。
- access token。
- webhook secret。
- 原始客户聊天记录。
- 完整手机号、邮箱、身份证号。
- 未脱敏知识库敏感段落。

## 6. 准确率下降排查 SOP

触发条件：

- 转人工率突然升高。
- 坐席改写率突然升高。
- 知识缺口增加。
- 客户反馈回答不准。
- 题库回归下降。
- 模型 provider 失败或延迟升高。

排查顺序：

1. 确认最近是否变更知识库。
2. 确认最近是否变更模型、prompt、检索 top-k 或 reranker。
3. 检查固定题库回归指标。
4. 抽样查看最近 20 条被改写草稿。
5. 检查知识缺口 Top 20。
6. 检查渠道来源是否出现新场景。
7. 检查是否有新活动、新价格、新售后政策。
8. 修订知识库。
9. 重新跑回归。
10. 输出复盘和后续优化计划。

严禁：

- 只调大模型温度。
- 只换更贵模型。
- 不看引用证据就判断模型问题。
- 不跑题库就发布知识库。

## 7. 知识库更新 SOP

流程：

1. 客户提交变更需求。
2. 我方确认资料来源和生效时间。
3. 整理为知识卡片或文档版本。
4. 标记高风险和禁用话术。
5. 在测试环境或 staging 索引中导入。
6. 跑固定题库。
7. 抽查 AI 草稿。
8. 客户确认口径。
9. 发布。
10. 保留回滚版本。

每次知识库变更必须记录：

- 变更人。
- 审核人。
- 生效时间。
- 影响范围。
- 相关题库。
- 回归结果。
- 回滚方式。

## 8. 故障分级 SOP

| 等级 | 情况 | 内部动作 |
| --- | --- | --- |
| P1 | 系统不可用、误外发风险、数据恢复 | 立即止血，关闭外发，保留现场，拉工程处理 |
| P2 | 模型不可用、渠道大面积失败、知识库错误发布 | 切 fallback，暂停自动化，通知客户 |
| P3 | 单功能异常、部分账号异常、质量下降 | 排期排查，输出修复计划 |
| P4 | 文案、配置、培训问题 | 客户成功处理 |

P1/P2 必须复盘：

- 发生原因。
- 影响范围。
- 响应时间。
- 修复动作。
- 是否需补测试。
- 是否需更新 SOP。
- 是否需客户沟通模板。

## 9. 运维成本模型

内部估算不能只看服务器和 token。

成本项：

- 云服务器、数据库、对象存储、备份。
- 模型调用。
- 短信/通知/渠道服务费。
- 知识库整理人力。
- 题库标注人力。
- 坐席培训人力。
- 月度质量复盘人力。
- 故障响应人力。
- 渠道授权和平台排障人力。

建议内部报价底线：

| 项目 | Lite 试点版 | 标准运营版 |
| --- | ---: | ---: |
| 一次性交付内部人力 | 3-8 人日 | 15-40 人日 |
| 月维护内部人力 | 0.5-2 人日 | 3-8 人日 |
| 月云资源和模型成本 | 低到中 | 中到高 |
| 建议维护费底线 | 不低于 ¥1,500/月 | 不低于 ¥6,000/月 |

如果客户要求私有化、多渠道、专属 SLA、周报、驻场或高并发，必须单独报价。

## 10. 月度运维报告模板

每月内部先生成，再决定给客户输出哪一版。

内容：

- 本月咨询量。
- AI 草稿数量。
- 人工审核数量。
- 自动回复数量，如已开启。
- 转人工率。
- 知识命中率。
- 引用支撑率。
- 坐席改写率。
- 知识缺口 Top 10。
- 失败发送 Top 原因。
- 模型调用量和成本估算。
- 渠道健康。
- 本月知识库变更。
- 下月优化计划。

## 11. 续费和升级判断

适合从 Lite 升级标准运营版的信号：

- 咨询量稳定增长。
- 坐席超过 3 人。
- 知识缺口持续增加。
- 客户要求更多渠道。
- 客户要求质量报告和审计。
- 客户希望开放低风险自动回复。
- 客户需要团队分配、SLA、工单或绩效。

适合专项报价的信号：

- 私有化部署。
- 高并发。
- 多渠道官方接入。
- 电商平台服务商授权。
- 本地模型。
- 专属 SLA。
- 合规审计。

## 12. 内部红线

禁止：

- 未授权真实外发。
- 未脱敏真实客户数据进入外部模型。
- 把模拟发送说成真实发送。
- 把官网沙盒说成全平台接通。
- 用客户生产数据做随手测试。
- 在聊天、文档或日志中保存密钥。
- 为了成交承诺 100% 准确率。
- 用个人微信外挂、Hook、群控作为正式方案。

## 13. P3-05B 托管云与私有化交付补充

托管云端版和私有化部署都必须先按 Lite 试点版的安全边界推进：单客户单实例、真实外发默认关闭、客户知识和题库先脱敏、远程维护临时授权、所有高风险动作留痕。

### 13.1 托管云端版

| 项目 | 内部要求 |
| --- | --- |
| 部署形态 | 首批只按单客户单实例交付，不包装成成熟多租户 SaaS |
| C0 试点 | 可用云主机 + Docker Compose + PostgreSQL/Redis 容器 + HTTPS 反向代理 |
| C1 正式 | 需要升级为版本化镜像、独立 PostgreSQL/Redis、集中日志、备份恢复和回滚 |
| 外发 | `OUTBOX_EXTERNAL_WRITE_ENABLED=false`，真实外发另行授权 |
| 前端 | 当前容器仍是试点形态，正式托管前需评估静态化/Nginx/CDN |
| 模型 | 真实 key 只在客户授权后配置，小样本 smoke 必须限量 |

### 13.2 私有化部署

| 项目 | 内部要求 |
| --- | --- |
| 交付包 | 必须包含镜像、compose、env 模板、迁移说明、备份恢复、诊断脚本、客户资料和 release notes |
| 客户准备 | 服务器、域名/证书、防火墙、备份目录、管理员、模型 key、平台授权、远程维护通道 |
| 安装验收 | 健康检查、登录、知识导入、文档检索、AI 草稿、人审、outbox 门禁、诊断包 |
| 权限回收 | 安装完成后回收临时账号、临时隧道和临时密钥 |
| 禁止 | 长期保存客户密码、明文传 key、无备份迁移、未授权真实外发 |

### 13.3 内部交付底线

- `P3-05B_HOSTED_CLOUD_RUNBOOK.md`

- `REMOTE_MAINTENANCE_AUTHORIZATION_SOP.md`
- P3-05B readiness smoke
- 客户确认的知识包和题库
- 部署、备份、回滚和权限回收记录
