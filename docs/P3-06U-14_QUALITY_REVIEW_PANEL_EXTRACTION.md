# P3-06U-14 质量复盘页组件拆分

日期：2026-07-02

## 1. 工程控制卡

| 项目 | 内容 |
| --- | --- |
| 本片目标 | 把质量复盘页从 `App.tsx` 抽离，降低主应用文件体积，让后续前端优化更容易定位。 |
| 客户可见价值 | 页面视觉和行为保持不变，但后续继续优化质量 BI、错因下钻、知识修复闭环时不会继续挤压主壳层。 |
| 是否只是评测 | 否。本片是前端结构性改造，同时补静态门禁。 |
| 停止条件 | TypeScript、Vite build 和 P3-06U-14 静态门禁通过；不打开真实外发。 |
| 写回位置 | 本文档、`scripts/check_p3_06u_14_quality_panel_extraction.py`、Project_012 执行记录和文件索引。 |

## 2. 本轮改动

### 2.1 新增质量页组件目录

新增：

`frontend/src/components/quality/QualityReviewPanel.tsx`

承载原质量复盘页的主要结构：

- 错因到知识修复闭环。
- 质量复盘摘要。
- 错因排行。
- 质量漏斗。
- 题库趋势。
- 渠道异常分布。
- 知识缺口热力。
- 人审矩阵。
- 样本下钻。

### 2.2 主应用文件减负

`App.tsx` 不再定义 `QualityReviewPanel` 和质量页专用类型，只负责传入会话、人审、待发送、失败复盘、队列、知识评测和知识缺口数据。

`QualityMetric` 仍被知识评测报告复用，因此从质量组件文件作为命名导出提供。

### 2.3 行为边界

本片坚持“行为不变，结构变清楚”：

- 没有新增后端接口。
- 没有新增真实渠道连接。
- 没有新增真实外发。
- 没有改变质量 BI 的统计口径。
- 仍明确提示准确率必须结合人工标签，不能把检索命中率当作完整准确率。

## 3. 验收方式

| 验收项 | 方式 |
| --- | --- |
| 组件抽离 | `App.tsx` 只 import，不再定义 `QualityReviewPanel`。 |
| 质量页关键结构 | 静态门禁检查 `quality-repair-map`、`quality-bi-grid`、样本下钻和 smoke 标识。 |
| 类型契约 | `npm run typecheck`。 |
| 生产构建 | `npm run build`。 |
| 回归边界 | 不改真实外发开关，不改渠道 connector 行为。 |
