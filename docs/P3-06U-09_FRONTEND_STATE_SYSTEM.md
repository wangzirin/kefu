# P3-06U-09 前端状态体系统一

## 目标

本轮把中台里分散的提示语统一为同一套状态系统，让使用者能快速判断当前页面处于哪一种状态：

- 加载中。
- 暂无数据。
- 无权限。
- 配置缺失。
- 接口失败。
- 演示样本。
- 真实服务端数据。
- 真实外发关闭。

这不是单纯换样式，而是把“数据从哪里来、为什么按钮不能点、现在会不会真实发消息”讲清楚。

## 已完成内容

### 1. 统一状态组件

在 `frontend/src/App.tsx` 中新增：

- `WorkspaceStateNotice`：统一加载、空态、无权限、配置缺失、接口失败等提示。
- `PanelStateNotice`：把各个页面的 `idle / loading / ready / error` 状态转换为统一提示。
- `DataSourceBadge`：统一展示“演示样本 / 真实服务端数据 / 配置缺失 / 真实外发关闭”。
- `DisabledReason`：统一解释禁用按钮为什么不可用。
- `WorkspaceRuntimeStateStrip`：在工作台顶部持续展示当前数据来源、配置状态和外发边界。

### 2. 核心页面落地

已覆盖核心客服链路：

- 顶部全局状态条：始终展示数据来源与真实外发边界。
- 多渠道对话台：统一“演示样本 / 真实服务端数据 / 真实外发关闭”，补充批准按钮禁用原因。
- 人工审核收件箱：统一加载、权限、空态、外发门禁和禁用原因。
- 待发送草稿：统一 outbox 与发送队列状态，明确“审核通过不等于已外发”。
- 失败复盘队列：统一回执异常与复盘状态，明确不会自动重发。
- 渠道连接器中心：统一官方渠道配置缺失、回调卡点和真实外发关闭。
- 知识文档运营：统一知识数据来源、检索口径、空文档、空检索结果和导入禁用原因。
- 知识缺口闭环：统一缺口同步、修复状态和外发边界。
- 知识评测与质量：统一题集、运行历史、质量口径和空运行状态。
- 质量诊断 BI：统一质量口径，明确当前 BI 不把检索命中当作完整准确率。
- 运维与告警：统一 worker、告警、指标、风险与运行记录空态。

### 3. 样式体系

在 `frontend/src/styles.css` 中新增：

- `.workspace-state-ledger`
- `.panel-state-row`
- `.data-source-badge`
- `.workspace-state-notice`
- `.workspace-state-icon`
- `.disabled-reason`

状态色保持克制，不做花哨卡片：

- 蓝色：加载、说明、本地检索口径。
- 绿色：真实服务端数据。
- 橙色：演示样本、配置缺失、无权限、真实外发关闭。
- 红色：接口失败。
- 虚线：暂无数据。

## 产品边界

当前统一的是前端状态表达，不改变后端权限、渠道连接、知识检索和真实外发逻辑。

真实外发仍保持关闭：页面上所有“发送、队列、渠道计划、复盘”都只代表内部状态，不能被解释为已经向微信、企微、抖音、淘宝等平台真实发送。

## 验收方式

新增静态检查：

```bash
python3 scripts/check_p3_06u_09_unified_states.py
```

新增浏览器烟测：

```bash
CHROME_DEBUGGING_URL=http://127.0.0.1:9227 \
FRONTEND_URL=http://127.0.0.1:5181/ \
node scripts/check_p3_06u_09_unified_states.mjs
```

浏览器烟测会检查：

- 顶部状态条存在。
- 演示样本、配置缺失、真实外发关闭可见。
- 核心页面存在统一状态组件或数据来源标签。
- 多渠道对话台、知识库、质量 BI、待发送、渠道中心、运维页无横向溢出。
- 关键页面截图写入 `output/p3_06u_09_unified_states/`。
