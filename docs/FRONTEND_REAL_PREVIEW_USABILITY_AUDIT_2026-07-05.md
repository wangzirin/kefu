# 前端真实预览可用性审查记录 2026-07-05

## 结论

当前前端已经可以作为本地受控试跑界面使用：本地服务可启动，登录页可进入，运营总览、多渠道对话台和渠道接入页能读取后端数据并完成基础操作。

但它还不是成熟全渠道商用版本。真实客户资料、真实平台入站/外发、正式客户签收、签名安装器和生产 SLA 仍未闭环。

## 本轮修复

- 登录页新增“进入本地预览”按钮，仅在开发入口开启时显示，调用已有 `/api/auth/dev-local-login`，解决本地预览卡在未知密码的问题。
- 多渠道对话台请求增加 `channel_id` 支持，从总览按渠道进入时由后端筛选分页数据，不再只依赖前端当前页过滤。
- 渠道接入页清理客户可见工程词，将 `channel_accounts`、`Hook`、`dry-run` 等表达替换为客户可理解的业务语言。

## 已验证

- `npm run typecheck`
- `npm run build`
- `node scripts/check_p3_06u_26h2w_fe7_customer_trial_browser_smoke.mjs`

## 仍需继续优化

- 真实客户资料链路仍处于等待态，不能写成客户数据试跑完成。
- 渠道页虽然边界清楚，但仍偏实施后台，需要进一步降低技术参数首屏权重。
- 工作台在真实空数据场景下需要更强的下一步引导。
- 当前为本地开发预览，`dev_bootstrap_enabled=true`，不是正式客户封包状态。
