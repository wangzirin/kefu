# P3-06U-08 渠道连接器中心实用化

| 项目 | 内容 |
| --- | --- |
| 阶段编号 | P3-06U-08 |
| 阶段名称 | 渠道连接器中心实用化 |
| 施工目标 | 让客户和交付人员一眼看懂每个渠道卡在哪一步、为什么不能自动回复、下一步要补什么配置 |
| 本片性质 | 前端状态中心产品化，不新增后端字段，不打开真实外发 |
| 当前边界 | 企业微信/微信客服优先展示完整接入路线，其它平台只展示官方授权前置条件和未接入状态 |

## 本片解决的问题

此前 `#channels` 页已经有渠道健康表、官网沙盒和失败复盘，但客户进入页面后仍然很难回答几个关键问题：

1. 企业微信/微信客服现在到底卡在创建客服、绑定接待人员、回调 URL、可信 IP、URL 验证还是白名单发送。
2. 公网 HTTPS 回调、Token、EncodingAESKey 应该填在哪里，哪些信息不能在前端明文展示。
3. 公众号、抖音、小红书、淘宝、京东、拼多多是否已经接通，还是仍然需要官方授权。
4. 为什么现在不能直接自动回复真实客户。

本片把渠道页改成“状态中心”：先显示主通道接入进度，再显示敏感配置状态、正式商用阻塞条件和其它平台官方授权前置条件。

## 已完成改动

- 新增 `ChannelConnectorCenterPanel`，作为 `#channels` 页面第一屏。
- 企业微信/微信客服展示 10 个接入状态：
  - 未配置。
  - 已创建客服账号。
  - 已绑定接待人员。
  - 已获取链接/二维码。
  - 回调 URL 待配置。
  - URL 验证通过。
  - 已收到入站消息。
  - 已生成 AI 草稿。
  - 已进入人工审核。
  - 白名单发送测试通过。
- 显示当前主卡点，例如“回调 URL 待配置”或“等待白名单发送测试”。
- 显示公网 HTTPS 回调 URL、Token、EncodingAESKey、可信 IP 的配置状态；Token 和 EncodingAESKey 只显示 secret 引用，不展示明文。
- 正式商用前置条件写明：已备案域名、公网 HTTPS、可信 IP、密钥服务端存储、URL 验证、AES 解密、XML 解析、入站幂等、人工审核和白名单测试。
- 公众号、抖音/抖店、小红书、淘宝/天猫、京东/拼多多只展示“未接入”和官方授权前置条件，不暗示可以通过外挂、Hook、模拟点击、个人号群控或商家后台 RPA 交付。

## 工程边界

- 没有新增真实渠道配置表或 secret store 字段。
- 没有读取或展示真实 Token、EncodingAESKey、Secret 明文。
- 没有打开真实外发。
- 没有完成企业微信公网 HTTPS 回调 smoke。
- 没有完成公众号、抖音、小红书、淘宝、京东或拼多多真实平台授权。
- 没有把白名单测试包装成全量自动回复上线。

## 验收标准

- `#channels` 页面第一屏出现“渠道接入状态中心”。
- 企业微信/微信客服主卡片出现 10 个接入状态。
- 页面必须出现“回调 URL 待配置”“URL 验证通过”“已收到入站消息”“已生成 AI 草稿”“已进入人工审核”“白名单发送测试通过”。
- 页面必须出现“公网 HTTPS 回调 URL”“Token”“EncodingAESKey”“可信 IP”，但不得出现真实密钥明文。
- 页面必须说明“已备案域名”“公网 HTTPS”“可信 IP”“不自动真实外发”等正式商用阻塞条件。
- 公众号、抖音/抖店、小红书、淘宝/天猫、京东/拼多多必须显示官方授权前置条件和未接入状态。
- 桌面、窄桌面和移动端没有横向溢出。

## 后续

P3-06U-09 进入前端状态体系统一。渠道中心后续应继续接入真实后端配置状态、企业微信公网 HTTPS 回调 smoke、可信 IP 检查、access token 状态、官方回执健康和白名单发送验收，但这些都需要客户真实后台权限、已备案域名或公网测试隧道，以及用户明确授权。
