## 4. 全局硬门禁

### 4.1 全局验收门禁

整个 H2W 路线最终必须同时满足：

1. 客户可见页面没有假按钮、假图标、假筛选、假导出、假上传、假发送。
2. 每个写动作都有前端动作、API client、后端接口、权限校验、数据变化、失败反馈和审计证据。
3. 每个读动作都有真实接口或明确本地兜底说明，筛选、分页、排序口径可验证。
4. 多渠道对话台像真实客服软件，而不是内部工程面板。
5. 知识建设中心能让中小企业客户自己录入、预检、发布和复盘知识。
6. 质量复盘能定位错因，并能跳转到知识修复或自动回复策略调整。
7. 报告、签收、诊断包、更新包、恢复包都有版本、时间、操作者、租户和可回溯记录。
8. 真实外发关闭时，任何页面不得暗示已发到外部平台。
9. 浏览器 smoke 覆盖登录、目标页、关键按钮、关键表单、关键状态变化。
10. 阶段完成后写回 Project_012 的 `执行记录.md`、`文件索引.md`、`关键决策.md`、`复盘与采坑.md`。

### 4.2 全局停止门禁

出现以下任意情况，必须停止当前目标，不允许继续堆新功能：

1. 客户可见按钮点击后无响应，且没有禁用说明。
2. 页面出现“已自动发送、已接通全渠道、已完成线上准确率、已上传云端、已正式签收、程序包可直接应用”等超过真实能力的文案。
3. 前端存在写动作，后端没有接口。
4. 后端有接口，前端按钮没有接入，却对客户展示为可用。
5. 写动作没有权限校验、幂等保护或审计记录。
6. 筛选按钮只改变选中样式，不改变数据口径，也没有兜底说明。
7. 浏览器 smoke 只截首页，不覆盖关键按钮和关键页面。
8. 使用演示样本冒充真实客户数据。
9. 把 dry-run 写成已执行。
10. 把 draft-only 写成已发送。
11. 把检索评测写成完整客服准确率。
12. 把本地确认记录写成正式电子签章。
13. 把 RPA 研究线写成正式默认交付。
14. 没有项目写回记录却声称阶段完成。

## 7. H2W-2：客户知识建设中心

### 7.1 目标

让中小企业客户能自己维护客服知识：录入业务对象、标准问答、流程政策、禁用承诺和转人工规则，经过预检、样题试跑、发布确认后生效。

### 7.2 四层知识模型

| 层级 | 客户理解 | 后端对象 | 用途 |
|---|---|---|---|
| 业务对象 | 产品、服务、门店、套餐、活动 | `business_objects` | 判断客户问的是什么 |
| 标准问答 | 高频问题、相近问法、标准答案 | `object_knowledge_cards` 或知识卡 | 生成可引用回答 |
| 流程政策 | 售后、退款、发票、物流、保修、预约 | policy 类知识卡或扩展字段 | 处理流程类问题 |
| 禁用承诺与转人工规则 | 不能承诺、必须人工确认、投诉赔付 | reply policy/rules | 阻断风险回复 |

### 7.3 施工范围

- [ ] 知识库运营页提供四层入口。
- [ ] 新增业务对象可点击、可提交、可落库、可刷新。
- [ ] 标准问答可绑定业务对象。
- [ ] 流程政策有分类、适用范围、生效状态。
- [ ] 禁用承诺和转人工规则能被自动回复策略读取，或明确进入待接入清单。
- [ ] 支持客户导入 50-100 条脱敏问题和知识包模板。
- [ ] 发布前走预检和样题试跑。
- [ ] 发布后记录版本、发布人、发布时间和影响范围。

### 7.4 验收门禁

1. 新增业务对象按钮可点击，并能真实保存。
2. 编辑业务对象后刷新页面仍能读取最新值。
3. 新增问答卡能绑定业务对象，并能在列表看到。
4. 导入模板 dry-run 能显示新增、跳过、错误和风险。
5. 知识评测能读取题库或样题，输出发布前结论。
6. 禁用承诺和转人工规则至少能进入自动回复策略配置，不能只停留在文案。
7. 所有客户可见错误提示能被非工程用户理解。

### 7.5 停止门禁

## 8. H2W-3：线上回执与准确率闭环

### 8.1 目标

建立“问题 -> 回答 -> 人工标签 -> 错因 -> 知识修复 -> 回归验证”的质量闭环。真实平台回执未完成前，指标只能叫“本地样本质量”或“人工标签质量”。

### 8.2 施工范围

- [ ] 支持最终回复样本导入和采样。
- [ ] 支持人工事实性标签：正确、部分正确、错误、需人工、禁用承诺风险。
- [ ] 支持错因分类：无知识、知识过旧、引用不足、策略错误、模型误判、渠道异常。
- [ ] 质量复盘页展示准确率、转人工率、错因分布、修复后变化。
- [ ] 错因能跳转到知识缺口或自动回复策略。
- [ ] 修复后能运行回归题集并比较前后变化。

### 8.3 验收门禁

1. 能导入或采样最终回复样本。
2. 人工标签能保存到后端。
3. 质量复盘能读取标签结果。
4. 每个错误样本能进入错因分类。
5. 错因能生成知识修复任务或策略调整建议。
6. 修复后能运行回归题集并比较前后变化。

4. 把本地样本质量写成线上全量准确率，停止。

### 9.3 验收门禁

3. 导出文件能打开，格式正常。
4. 报告指标和系统页面口径一致。
5. 签收记录能列表查看、详情查看、按版本追溯。
6. 导出文件不包含 secrets、真实 token、密码、私钥。

### 9.4 停止门禁

1. 只生成 HTML 预览，却声称支持 PDF/DOCX，停止。
2. CSV 被写成 XLSX，停止。
3. 签收记录没有版本和适用范围，停止。
4. 本地确认被写成正式电子签章，停止。
5. 导出文件含敏感凭据，停止。

## 12. H2W-7：生产级 RAG 与模型成本治理

### 12.1 目标

把“能回答”升级为“可解释、可控成本、可回归、可降级”的客服知识检索和模型调用体系。

### 12.2 施工范围

- [ ] 生产级检索路线：关键词、向量、混合检索、重排、引用溯源。
- [ ] 每次回答记录引用来源、置信度、转人工理由、模型调用成本。
- [ ] 高置信 FAQ 不调模型。
- [ ] 普通问题走默认模型。
- [ ] 高风险、低置信、无知识、禁用承诺命中时转人工。
- [ ] 模型预算：日预算、月预算、单次上限、降级策略。
- [ ] 对客户显示为“自动回复策略”，不直接暴露复杂模型路由术语。

### 12.3 验收门禁

1. 每条自动回复都有引用来源或无引用原因。
2. 每条模型回复都有模型名、token 或费用估算。
3. 超预算后能降级或转人工。
4. 高风险问题不会自动承诺。

7. 模型成本不只存在日志里，必须能被质量复盘或自动回复策略页面读取。

### 12.4 停止门禁

1. 无引用却高置信回答，停止。
2. 没有成本记录却展示成本治理，停止。
3. 高风险问题仍自动外发，停止。
4. 客户界面只写模型名，不解释策略，停止。
5. 把模型回答质量当成知识库质量，停止。
