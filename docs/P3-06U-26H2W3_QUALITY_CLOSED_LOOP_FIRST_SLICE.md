# P3-06U-26H2W3 线上回执与准确率闭环第一片

## 目标

本片把已有的最终回复样本、人工事实性标签、月度质量复盘、客户质量报告和知识修复入口串成一条可操作闭环：

渠道入站问题 -> 最终客服回复样本 -> 人工事实性标签 -> 错因归类 -> 知识修复 / 自动回复策略调整 -> 同题集回归验证。

当前仍不打开真实平台外发，不声明线上全量准确率。真实平台回执未接入前，所有质量口径都必须写成“本地样本质量”“人工标签质量”或“受控试点质量”。

## 本片完成内容

1. 质量复盘页新增 `H2W-3 线上回执与准确率闭环` 区域。
2. 前端明确区分六个状态：最终回复样本、人工事实性标签、错因定位、知识修复入口、发布后回归、线上回执。
3. 人工标签结果拆成正确、部分正确、错误、需人工四类，不再把转人工和错误混在一起看。
4. 错因修复入口可跳转到知识缺口、知识库运营、自动回复策略和知识评测页。
5. 页面显式写出：真实外发继续关闭；真实平台回执未接入前，不展示完整线上准确率。

## 验收门禁

- 能看到 `data-h2w3-quality-loop="true"`。
- 必须出现 `sample-to-label`、`cause-to-repair`、`post-publish-regression` 三个闭环步骤。
- 必须出现 `no-full-online-accuracy` 边界。
- 页面必须把检索命中率和最终客服答案正确率分开讲。
- 错因必须能跳转到知识缺口、知识库运营、自动回复策略或评测页。
- 真实平台外发、线上回执、失败重试、审计闭环仍需单独验收。

## 停止门禁

- 没有人工标签却展示完整准确率，立即停止。
- 错因只能看不能修，立即停止。
- 把检索命中率当成最终答案正确率，立即停止。
- 没有真实平台回执却写成线上全量准确率，立即停止。
- 质量图表无法下钻到样本或错因，立即停止。
