# P3-06U-10C 默认总览与侧栏收束

日期：2026-07-02

## 目标

本片只处理前端中台壳层和总览首屏，不改后端业务逻辑、不打开真实外发、不改渠道接入状态。

目标是让工作台更像可交付的商户客服中台：

1. 登录或开发演示进入后，默认落到“运营总览”。
2. 坐席也能看到总览，但日常处理仍从“工作台”进入。
3. 左侧导航收窄，一级菜单去掉副标题和右侧小标签。
4. 多入口工作域默认折叠，点击“工作台”等一级菜单后再展开二级入口。
5. 总览页首屏优先展示 BI 图表、健康度、趋势和风险信号，减少顶部解释性内容。

## 改动范围

| 文件 | 改动 |
| --- | --- |
| `frontend/src/data/navigation.ts` | 总览对 `agent` 可见；默认入口统一为 `#overview`。 |
| `frontend/src/App.tsx` | 登录和开发演示进入后落到总览；侧栏加入折叠状态；总览页不再显示任务路径、演示横幅和运行状态条。 |
| `frontend/src/styles.css` | 左侧栏从 280px 收到 224px；一级菜单去副标题；总览顶栏和 BI 头部压缩。 |
| `scripts/check_p3_06u_05_real_login_role_smoke.mjs` | 坐席真实登录默认页从 `#live` 更新为 `#overview`。 |

## 真实登录说明

当前前端有两种入口：

| 入口 | 含义 | 适用 |
| --- | --- | --- |
| 登录 | 使用 `tenant_slug + email + password` 调用 `/api/auth/login`，成功后保存 bearer token。 | 真实租户账号、真实后端数据、权限控制。 |
| 开发演示进入 | 无 token，调用 `/api/auth/me` 返回开发演示身份，并加载本地样例数据。 | 本地预览、演示 UI、流程验证。 |

真实登录不是“演示模式”。真实登录需要后端已启动、数据库有租户和用户，并且用户具备对应角色。演示模式只用于看界面和流程，不代表真实渠道、真实知识库或真实外发已启用。

## 验收标准

- 打开本地预览后，点击“开发演示进入”进入 `#overview`。
- 总览页首屏不出现顶部任务路径条、演示横幅和运行状态条。
- 侧栏宽度不超过 240px。
- “工作台”二级入口默认不展开，点击后显示多渠道对话台、会话收件箱、人工审核、待发送草稿和工单/SLA。
- 总览页首屏能看到 BI 信号卡、运营健康、趋势图或优先动作之一。

## 边界

- 本片不删除任务路径组件，只是不在总览页首屏展示。
- 本片不改变 RBAC 后端权限。
- 本片不启用真实平台外发。
- 本片不处理企业微信可信 IP、域名备案或回调 URL 配置。
