## 本阶段完成内容

### 后端接口

新增正式会话收件箱接口：

```text
GET /api/tenants/{tenant_id}/conversation-inbox
```

支持：

- 按租户读取会话。
- Bearer Token 登录校验。
- `status`、`channel_id`、`priority`、`assigned`、`sla`、`query`、`sort`、`page`、`page_size`。
- 返回服务端分页结构：`items`、`page`、`page_size`、`total`。
- 每条会话聚合客户、渠道、最后消息、等待时间、SLA 状态、人工审核任务数、待发送草稿数、发送失败复盘数和下一步动作。

新增坐席分配/接管接口：

```text
PATCH /api/conversations/{conversation_id}/assignment
```

支持：

- 将会话分配给当前坐席或指定坐席。
- 可同时调整团队和状态。
- 校验租户隔离，跨租户会话返回 404。
- 写入会话事件，保留审计链。

### SLA 轻量骨架

当前 SLA 是轻量计算，不是完整客服 SLA 系统：

- 最近一条客户入站消息后，如果没有更新的出站消息，则视为等待处理。
- 等待超过 30 分钟标记为 `warning`。
- 等待超过 60 分钟标记为 `breached`。
- 无客户等待时标记为 `idle`。

后续正式 SLA 还需要：

- 按客户套餐、工作时间、节假日、渠道优先级和会话类型配置不同 SLA。
- 支持暂停计时、恢复计时、升级提醒和超时归因。
- 将 SLA 事件写入审计和报表。

### 前端中控台

新增“会话收件箱”主面板：

- 当前会话、客户等待、SLA 超时、待审核指标。
- 搜索、状态筛选、每页数量和分页。
- 正式登录模式下读取后端分页、搜索、筛选和排序。
- 演示模式下使用 18 条样例会话，可验证分页和运营视图。
- 每条会话展示客户、会话主题、渠道、最后消息、SLA、等待时间、优先级、待审、待发、失败复盘、下一步动作和接管按钮。
- 演示模式下接管按钮禁用，避免误导为真实业务已接通。

## 当前不能承诺

- 不能承诺企业微信、公众号、抖音、小红书、淘宝、京东或拼多多已经真实自动回复。
- 不能承诺 SLA 是完整生产级 SLA 系统。
- 不能承诺多坐席排班、权限矩阵、团队工单和客户画像已经完整完成。
- 不能承诺当前演示数据来自真实客户或真实渠道。
- 不能承诺真实外发已打开；真实外发仍受总开关、渠道授权、可信 IP、白名单和人工审核控制。

## 下一步建议

### P3-05I：正式坐席工作流

目标是把“接管”从单按钮推进为完整坐席操作：

- 领取会话。
- 转派会话。
- 释放会话。
- 标记等待客户、已解决、需跟进。
- 记录处理备注。
- 按角色控制坐席、主管、管理员权限。

### P3-05J：联系人与线索画像

目标是让中控台不只是看消息，而是看客户关系：

- 联系人详情。
- 渠道身份绑定。
- 最近会话历史。
- 标签、意向、来源、跟进状态。
- 留资线索和销售跟进状态。

### P3-05K：知识缺口闭环

目标是把低置信、无引用、错误回复和人工改写沉淀回知识库：

- 会话中标记知识缺口。
- 生成待补知识任务。
- 知识审核通过后重新索引。
- 用题库回归验证是否修复。

### P3-05L：真实渠道入站 smoke

当前仍建议优先企业微信/微信客服：

- 准备公网 HTTPS 回调。
- 企微后台填写 URL、Token、EncodingAESKey。
- 通过 URL 验证。
- 个人微信发测试消息。
- 系统创建可信入站消息。
- AI 草稿进入人工审核。
- 外发继续关闭，等白名单和可信 IP 准备完成后再做真实发送。
