# P3-06U-07 知识运营台产品化

| 项目 | 内容 |
| --- | --- |
| 阶段编号 | P3-06U-07 |
| 阶段名称 | 知识运营台产品化 |
| 施工目标 | 让知识负责人能从质量错因进入补知识、草稿编辑、发布前回归、发布和回滚 |
| 本片性质 | 前端工作流产品化，不新增后端字段，不打开真实外发 |
| 当前边界 | 结构化编辑清单写入纯文本文档正文；后续可再拆成正式字段 |

## 本片解决的问题

P3-06U-06 已经把质量复盘变成“错因 -> 修复路径”的入口，但知识运营页仍然偏像功能集合：文档、缺口和评测分在不同页面，客户需要自己理解先后关系。

本片把已有后端能力重新组织成一条知识运营工作流：

1. 从质量复盘或人审发现错因。
2. 进入知识缺口，生成知识草稿。
3. 编辑知识草稿，明确适用问题、标准答案、禁止承诺、引用来源、生效范围、版本和审核状态。
4. 加入回归题并运行发布前回归。
5. 发布知识或在出问题时回滚为草稿。

## 已完成改动

- 新增 `KnowledgeOperationsFlowPanel`，在 `#knowledge`、`#gaps`、`#evals` 三个知识相关页面顶部展示同一条知识运营流程。
- 任务上下文来源新增 `knowledge`，从知识运营流程跳转到目标页时显示“来自知识运营”，并保留任务、状态、时间窗口和匹配数量。
- 流程阶段包含：发现错因、补知识草稿、编辑与审核、发布前回归、发布与回滚。
- 发布前回归门禁明确展示：缺口已定位、草稿已生成、回归题已加入、门禁无阻断、可回滚。
- 回归影响预估明确区分“预计会变好”和“可能会变差”，避免把单次命中率包装成完整准确率。
- 知识草稿编辑表单新增发布前检查清单：适用问题、标准答案、禁止承诺、引用来源、生效范围、版本和审核状态。

## 工程边界

- 没有新增后端字段。当前仍把结构化知识内容写入 `raw_text`，通过纯文本模板约束运营人员填写。
- 没有声称完整知识版本 diff 已完成。当前回滚能力是回滚为草稿并保留审计记录，不恢复旧正文。
- 没有打开真实外发。知识发布只影响内部检索状态，不代表消息已经发往微信、企微或电商平台。
- 没有声称真实准确率达标。当前只展示检索命中、引用覆盖、期望词覆盖、需复盘等内部质量信号。

## 验收标准

- `#knowledge` 页面出现知识运营流程、发布前回归门禁、回归影响预估和版本状态。
- `#gaps` 和 `#evals` 页面顶部同样出现知识运营流程，证明三段流程不是孤立页面。
- 知识运营流程至少包含 5 个阶段，并且链接带 `from=knowledge`。
- 点击流程中的处理缺口入口后，目标页出现“来自知识运营”和“当前匹配”。
- 知识草稿编辑区清楚展示 6 项发布前检查清单。
- 桌面、窄桌面和移动端没有横向溢出。

## 2026-07-02 后续修正：P3-06U-26G4

P3-06U-07 的共享流程面板在早期用于说明知识运营闭环，但进入 P3-06U-26G4 后确认它会让知识库运营、知识缺口、知识评测三个入口看起来过于相似，削弱三页独立工作区的实际使用感。

- 知识库运营、知识缺口、知识评测分别承担独立职责，不再重复展示同一套流程总览。
- 后续校验以 `P3-06U-26G4_KNOWLEDGE_OPS_ALIGNMENT_AND_DEDUP_AUDIT.md` 为准。

本修正不打开真实外发，不改变后端数据模型，只把前端从“流程说明型页面”收束为三页独立工作区。
