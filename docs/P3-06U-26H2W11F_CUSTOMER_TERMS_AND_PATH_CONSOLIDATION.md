# H2W-11F 前端客户术语与知识维护路径收束

## 定位

H2W-11F 接在 H2W-11E 之后，只做客户可见语言和知识维护路径的最后收束。

本片不新增真实外发、不接真实渠道、不调用真实模型、不新增后端能力、不新增空按钮，也不把本地确认写成正式验收。目标是让小微企业客户第一次进入知识运营页时，看见的是一条能理解、能点击、能落库的知识维护路径，而不是一组工程阶段名、重复流程卡或内部术语。

## 本片完成项

1. 客户主流程改名。
   - 旧口径：客户知识发布闭环。
   - 新口径：知识维护总流程。
   - 新展示：整理资料 -> 检查 -> 导入 -> 启用 -> 复测 -> 确认。

2. 客户动作改名。
   - 主路径明确为：生成资料包 -> 检查资料包 -> 导入知识库。
   - 转换客户资料 -> 生成资料包。
   - 预检更新包 -> 检查资料包。
   - 导入本地知识库 -> 导入知识库。
   - 发布前试跑 -> 启用前复测。
   - 确认发布 -> 启用知识。
   - 进入知识评测 -> 查看复测题库。

3. 客户资料入口改名。
   - 旧口径：客户资料导入助手。
   - 新口径：客户资料整理。
   - 资料进入系统后生成知识资料包，不再在客户可见界面反复强调 JSON、更新包和预检差异。

4. 资料包导入口径收束。
   - 旧口径：知识更新包导入。
   - 新口径：知识资料包导入。
   - 保留真实能力：检查资料包、导入资料包、展示新增/跳过/错误和导入批次。

5. 边界文案收束。
   - 继续明确真实外发关闭。
   - 把“本地签收记录不是正式电子签章”在知识维护主流程里改成更自然的“本地确认记录不是正式验收”。
   - 质量报告页仍可保留电子签章边界，因为它确实涉及报告确认和留档。

## 保留的真实功能

- 生成资料包仍调用 `applyCustomerKnowledgeIntakePackage`。
- 检查资料包仍调用 `onPreviewUpdatePackage` 和后端资料包预检接口。
- 导入知识库仍调用 `onImportUpdatePackage` 和后端资料包导入接口。
- 启用前复测仍调用 `onCheckPublishDocument(firstReadyDocument)`。
- 启用知识仍调用 `onPublishDocument(firstReadyDocument)`。
- 查看复测题库仍跳转 `#evals`。
- 查看质量报告仍跳转 `#quality`。

## 停止门禁

出现任一情况，本片不能通过：

- 客户主流程区域仍出现“客户知识发布闭环、预检更新包、转换客户资料、发布前试跑、确认发布、进入知识评测”等旧词。
- 客户资料整理区域仍出现“客户资料导入助手、转换为更新包、预检差异”等旧词。
- 资料包导入区域仍出现“知识更新包导入、导入更新包、更新包 JSON”等客户不需要理解的词。
- 新文案只改了文字，但按钮没有继续绑定真实 handler。
- 浏览器验收不能通过真实登录表单进入知识运营页。
- 点击生成资料包、检查资料包、导入知识库后，同一租户后端没有业务对象、知识文档或评测集落库证据。
- 页面出现“真实外发已开启、已接通全渠道、正式电子签章已完成、合同签收已完成、完整线上准确率已完成”等越界表达。
- 把本地确认记录写成正式验收、合同签收或电子签章。

## 本片不做

- 不新增真实平台外发。
- 不接企业微信、公众号、抖音、淘宝、京东、拼多多真实渠道。
- 不调用真实大模型供应商。
- 不新增自动解析 PDF、DOCX、XLSX 的能力。
- 不新增正式电子签章。
- 不新增移动端验收。
- 不新增生产级向量库或完整线上准确率验收。

## 下一步

H2W-11F 通过后，下一阶段应继续围绕“客户真实资料 -> 知识资料包 -> 复测题库 -> 质量报告 -> 本地确认”的闭环推进，优先补真实客户脱敏样本、客户标准答案口径、质量报告确认和线上回执证据。真实 IM 和官方渠道外发仍需单独授权后再进入专项。
