# P3-05B 私有化部署包清单

更新时间：2026-06-29  
适用范围：Lite 试点版私有化、小型客户内网部署、标准运营版早期私有化评估  
状态：P3-05B 第二片；本文是交付包定义，不代表已在客户环境完成部署。

## 1. 交付原则

私有化部署不能只给源码或一份 Compose 文件。正式交付必须是“版本化部署包 + 安装手册 + 迁移/备份/回滚 + 诊断包 + 验收清单 + 权限回收”的组合。

默认边界：

- 一个客户一个实例。
- 默认不真实外发。
- 默认不接个人号外挂。
- 默认不保存客户生产密钥。
- 默认不导入未脱敏客户聊天记录。
- 默认不承诺高并发 SLA。

## 3. 交付物清单

| 类别 | 文件/内容 | 是否必须 | 说明 |
| --- | --- | --- | --- |
| 应用镜像 | backend 镜像 | 是 | 后端 API |
| 应用镜像 | frontend 镜像 | 是 | 当前 P3-05B 仍是 C0 试点形态 |
| 编排文件 | `docker-compose.yml` | 是 | 基础服务定义 |
| 编排文件 | `docker-compose.pilot.yml` | 是 | 试点覆盖，包含健康检查、持久化 volume、外发关闭 |
| 环境模板 | `env.example` | 是 | 只放变量名和安全默认值 |
| 迁移说明 | migration runbook | 是 | 升级前备份、迁移后 smoke |
| 诊断脚本 | `create_p3_05_diagnostic_bundle.py` | 是 | 只读脱敏诊断 |
| 自检脚本 | `check_p3_05_deployment_readiness.py` | 是 | 文件和配置边界检查 |
| 客户资料 | 使用手册、产品介绍、服务体系 | 是 | 面向客户使用和试点沟通 |
| 备份说明 | 数据库和知识资料备份 | 是 | 必须含恢复演练要求 |
| Release notes | 版本、变更、兼容、风险 | 是 | 交付和升级追溯 |
| 校验值 | SHA256 | 建议必须 | 防止离线包传输损坏 |

## 4. 客户侧准备清单

| 准备项 | 责任方 | 要求 |
| --- | --- | --- |
| 服务器 | 客户 | Linux 服务器或客户容器平台 |
| CPU/内存 | 客户 | Lite 建议 4 vCPU、8GB 内存起步 |
| 磁盘 | 客户 | 建议 100GB 起步，数据库和备份分开规划 |
| Docker | 客户或我方协助 | Docker Engine + Docker Compose 插件 |
| 域名/证书 | 客户 | 内网域名或公网 HTTPS 域名 |
| 防火墙 | 客户 | 默认只开放 80/443；数据库和 Redis 不暴露公网 |
| 备份位置 | 客户 | 本机外的备份目录、NAS 或对象存储 |
| 管理员账号 | 客户 | 指定系统管理员和知识负责人 |
| 模型 key | 客户 | 如使用云模型，需明确授权和费用归属 |
| 平台授权 | 客户 | 企业微信、公众号、电商平台必须官方授权 |
| 远程维护通道 | 客户 | VPN、零信任、堡垒机或临时隧道 |
| 数据脱敏 | 客户和我方 | 真实题库、聊天记录、知识资料需先脱敏 |

## 5. 我方交付检查清单

### 5.1 安装前

| 检查 | 通过标准 |
| --- | --- |
| 版本确认 | 部署包版本、镜像 tag、文档版本一致 |
| 密钥边界 | env 模板无真实 key |
| 外发边界 | `OUTBOX_EXTERNAL_WRITE_ENABLED=false` |
| 数据库策略 | 已确认新装、升级或迁移 |
| 备份策略 | 已确认备份目录和恢复演练方式 |
| 远程授权 | 已确认授权人、时间窗口和访问方式 |

### 5.2 安装中

| 步骤 | 产物 |
| --- | --- |
| 导入镜像 | `docker images` 可看到指定 tag |
| 校验 compose | `docker compose config` 可解析 |
| 启动服务 | `docker compose up -d` 成功 |
| 迁移数据库 | Alembic 到 head |
| 健康检查 | 后端 `/health` 通过 |
| 登录检查 | 管理员可登录 |
| 外发检查 | 真实外发关闭 |

### 5.3 安装后

| 验收 | 通过标准 |
| --- | --- |
| 知识导入 | Lite 知识包导入成功 |
| 文档检索 | 3-5 个问题可检索引用 |
| AI 草稿 | 小样本草稿进入人审 |
| Outbox | 发送计划 blocked/no-op |
| 题库 | 50 条模板或客户脱敏题可运行 |
| 诊断包 | 可生成且不含密钥和原文 |
| 培训 | 管理员、坐席、知识负责人完成基础培训 |
| 交接 | 客户确认账号、备份、运维联系人和边界 |

## 6. 离线环境安装

离线客户需额外交付：

| 内容 | 说明 |
| --- | --- |
| 镜像 tar | `docker save` 导出的 backend/frontend/postgres/redis 镜像 |
| 系统依赖 | Docker 离线安装包由客户环境决定 |
| 校验文件 | 每个 tar 对应 SHA256 |
| 离线文档 | 不依赖外部链接的安装说明 |
| 模型路径 | 离线环境通常不能用云模型，需另行评估本地模型或专线 |

离线导入示例：

```bash
docker load -i images/backend.tar
docker load -i images/frontend.tar
sha256sum -c images/checksums.sha256
```

## 7. 环境变量处理规则

| 规则 | 说明 |
| --- | --- |
| 不交付真实 `.env` | 我方只交付模板，客户真实值由客户保存 |
| 不在聊天传 key | API key、token、secret 不通过普通聊天软件发送 |
| 不长期持有 root 密码 | 远程维护用临时授权 |
| 变更留痕 | 每次 env 变更记录变更人、时间、原因和回滚方式 |
| 外发默认关闭 | `OUTBOX_EXTERNAL_WRITE_ENABLED=false` 是私有化默认值 |

## 8. 备份、恢复和升级

### 8.1 备份要求

| 对象 | 频率 |
| --- | --- |
| PostgreSQL | 试点每日；正式按客户制度 |
| 知识资料 | 每次发布前后 |
| 题库和报告 | 每次评测后 |
| 配置变更记录 | 每次变更后 |
| 部署包版本 | 每次上线后归档 |

### 8.2 升级步骤

1. 客户确认升级窗口。
2. 生成升级前诊断包。
3. 备份数据库和知识资料。
4. 导入新镜像。
5. 执行 compose config。
6. 执行迁移。
7. 跑健康检查和登录检查。
8. 跑知识检索和 outbox 门禁检查。
9. 客户确认恢复业务。
10. 保留旧包直到观察期结束。

### 8.3 回滚条件

- 健康检查失败。
- 登录失败。
- 数据迁移异常。
- 知识检索不可用。
- Outbox 门禁异常。
- 题库回归严重下降。
- 客户确认不可接受的业务影响。

## 9. 远程维护交接

私有化部署后，远程维护不默认开放。客户需在需要时提供临时授权。

| 通道 | 适用 |
| --- | --- |
| VPN/零信任 | 长期服务客户 |
| 堡垒机 | 中大型客户 |
| 临时隧道 | 短时排障 |
| 诊断包 | 安全要求高或无法远程登录 |

必须记录：

- 故障单号。
- 授权人。
- 操作人。
- 授权窗口。
- 权限范围。
- 是否只读。
- 是否涉及生产数据。
- 操作记录。
- 权限回收记录。

## 10. 禁止项

| 禁止项 | 原因 |
| --- | --- |
| 长期保存客户服务器密码 | 安全和责任边界不可控 |
| 明文传输 API key/token | 容易泄露且无法审计 |
| 无备份执行迁移 | 失败后无法恢复 |
| 未授权真实外发 | 可能造成客户业务和平台风险 |
| 个人微信外挂/Hook/群控 | 不进入正式交付方案 |
| 把官网沙盒说成全平台接通 | 属于销售和交付误导 |
| 用生产数据随手测试 | 隐私和合规风险 |
| 诊断包包含 `.env` 明文 | 密钥泄露风险 |
