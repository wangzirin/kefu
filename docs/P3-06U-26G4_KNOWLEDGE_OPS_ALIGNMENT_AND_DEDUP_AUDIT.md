# P3-06U-26G4 知识运营前后端对齐与去重审验

日期：2026-07-02  
范围：知识库运营、知识缺口、知识评测三页；前端组件、后端接口、数据流、重复入口与当前边界。

## 一、结论

知识运营三页已经不是纯静态样子，后端存在三组真实能力：

- 知识库运营：业务对象、对象问答卡、文档导入、文档分块、发布检查、发布、回滚、检索。
- 知识缺口：缺口列表、缺口同步、状态更新、从缺口生成文档草稿、从缺口生成回归题。
- 知识评测：评测集、评测运行、运行详情、报告导出、答案质量门禁指标。

但此前前端存在一个明显结构问题：`KnowledgeOperationsFlowPanel` 被同时挂到知识库运营、知识缺口、知识评测三页，导致三个入口视觉上和信息上都像同一页。页头还存在“新建业务对象 / 同步缺口 / 运行题集”等跳转式按钮，其中部分只是回到当前页，不是真正动作。这会造成操作误解，也削弱三页分工。

本轮已将共享流程总览面板从三页移除，并去掉知识页页头的重复动作按钮。三页保留各自真实操作面板：知识库运营负责知识资产维护，知识缺口负责错因和修复任务，知识评测负责题集和质量门禁。真实外发继续关闭。

## 二、前后端对齐矩阵

| 前端页面 | 当前主要组件 | 对应后端/API | 对齐状态 | 说明 |
|---|---|---|---|---|
| 知识库运营 | `KnowledgeWorkspacePage mode="library"` + `KnowledgeDocumentsPanel` | `business-objects`、`knowledge-cards`、`knowledge-documents`、`chunks`、`publish-checks`、`publication`、`rollback`、`knowledge-document-searches` | 已对齐 | 能维护业务对象、对象问答卡、文档、片段和发布记录。 |
| 知识缺口 | `KnowledgeWorkspacePage mode="gaps"` + `KnowledgeGapPanel` | `knowledge-gaps`、`knowledge-gaps/sync`、`knowledge-gaps/{id}`、`document-drafts`、`regression-cases` | 已对齐 | 能把人审、评测、回复策略产生的问题转成修复任务。 |
| 知识评测 | `KnowledgeWorkspacePage mode="evals"` + `KnowledgeEvaluationPanel` | `knowledge-evaluation-sets`、`runs`、`run detail`、`report` | 已对齐 | 当前知识评测是检索评测，不是完整客服准确率。 |
| 质量复盘到知识缺口 | `QualityReviewPanel` 与缺口入口 | 后端缺口同步服务 | 部分对齐 | 当前可形成缺口，但仍需更强的真实客户题库和错因标注闭环。 |
| 向量/RAG 运维 | 前端未作为主入口 | `knowledge-vector-index/rebuilds`、`plans` | 后端先行 | 这是生产级 RAG 的底层能力，暂不放入中小企业日常知识运营主界面。 |

## 三、本轮发现的问题

### 1. 重复入口

问题：`KnowledgeOperationsFlowPanel` 同时出现在三页，包含“发现错因、补知识草稿、编辑与审核、发布前回归、发布与回滚”等总览信息。  
判断：这个面板适合作为早期工程说明，不适合作为三页常驻工作台。  
处理：已移除该组件、挂载和样式。后续如需要流程说明，放到内部文档，不放在客户日常操作页面。

### 2. 页头假动作

问题：三页页头的部分按钮只是跳回当前页面，例如知识库页的“新建业务对象”、缺口页的“同步缺口”、评测页的“运行题集”。  
判断：真正动作应该出现在对应面板的操作区域，页头只负责定位当前页面和展示数据状态。  
处理：已移除 `knowledge-page-actions`。知识库新增、缺口同步、评测运行仍保留在各自真实面板内。

### 3. 知识缺口筛选边界

问题：后端 `knowledge-gaps` 支持状态、严重度、来源类型等筛选；当前前端主要使用状态分页，搜索在已加载数据内本地过滤。  
判断：中小企业试点阶段可接受，但大数据量时会产生“当前页内搜索”的局限。  
后续：若进入真实客户数据，需把搜索词、严重度、来源类型全部下推到服务端查询。

### 4. 知识评测边界

问题：页面上容易把“检索命中、引用覆盖、期望词覆盖”理解为完整客服准确率。  
判断：当前知识评测是检索评测，不是完整客服准确率。完整准确率还需要最终答案事实性、禁用承诺、转人工正确性、业务口径一致性和人工标签。  
处理：保留边界文案和答案质量门禁，后续继续把评测升级为客服答案质量评测。

## 四、去重后的页面职责

### 知识库运营

保留：

- 业务对象管理。
- 对象级问答卡。
- 文档导入。
- 文档片段。
- 文档检索。
- 发布检查、发布、回滚记录。
- 字段编辑清单：适用问题、标准答案、禁止承诺、引用来源、版本和审核状态。

不再承载：

- 跨页面流程总览。
- 缺口错因地图。
- 评测运行列表。

### 知识缺口

保留：

- 缺口列表。
- 缺口状态。
- 错因地图：无知识命中、引用不足、期望词缺失、人工驳回。
- 从缺口生成知识草稿。
- 从缺口生成回归题。
- 缺口发布动作入口。

不再承载：

- 知识资产编辑主界面。
- 评测题集管理。
- 重复的发布前流程说明。

### 知识评测

保留：

- 评测集。
- 运行评测。
- 运行详情。
- 报告导出。
- 发布前后对比入口。
- 答案质量门禁：最终答案事实性、引用充分、禁用承诺、转人工正确性。

不再承载：

- 知识文档编辑。
- 缺口列表主操作。
- 跨页面流程总览。

## 五、已实施改动

- 从 `#knowledge`、`#gaps`、`#evals` 三个路由中移除 `KnowledgeOperationsFlowPanel`。
- 删除 `KnowledgeOperationsFlowPanel` 函数组件。
- 删除 `.knowledge-ops-flow`、`.knowledge-ops-stage-grid`、`.knowledge-ops-detail-grid` 等已废弃样式。
- 从 `KnowledgeWorkspacePage` 移除 `primaryAction`、`secondaryAction` 和 `knowledge-page-actions`。
- 更新 `scripts/check_p3_06u_07_knowledge_ops.py` 和 `scripts/check_p3_06u_07_knowledge_ops.mjs`，旧流程总览不再作为通过条件。
- 更新 `scripts/check_p3_06u_26d_knowledge_three_pages.py` 和浏览器检查，防止共享流程总览回归。
- 新增 `scripts/check_p3_06u_26g4_knowledge_ops_alignment_dedup.py`，专项检查前后端对齐和去重边界。

## 六、仍然保留的边界

- 真实外发继续关闭。
- RPA 研究线不进入正式默认交付链。
- 当前知识评测是检索评测，不是完整客服准确率。
- 当前搜索和筛选还未完全服务端化。
- 当前没有承诺生产级大规模 RAG 检索质量。
- 当前没有接入真实客户题库的持续回归体系。

## 七、下一步建议

### P3-06U-26G5：知识缺口筛选服务端化

把缺口搜索词、严重度、来源类型、状态组合全部下推到后端，避免大数据量时只在当前页内搜索。

### P3-06U-26G6：知识库运营真实编辑体验

把业务对象、问答卡、文档编辑从“列表加表单”优化为更清晰的对象详情页，减少一次性铺满。

### P3-06U-26G7：知识评测最终答案质量闭环

继续把评测从“检索评测”升级为“客服答案质量评测”，加入事实性、口径一致性、风险承诺、转人工正确性和人工复核标签。

### P3-06U-26H：部署运维收束

完成客户账号、部署包、诊断包、备份恢复、远程维护授权、告警和月度质量复盘文档与工具闭环。
