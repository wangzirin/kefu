# P3-06U-12D 多渠道对话台坐席判断区收束

日期：2026-07-02

## 1. 本片目标

把多渠道对话台中分散的判断信息收束到一个坐席可直接使用的区域。

本片之前，`#live` 同时存在：

- 会话上下文快照。
- 回复策略摘要。
- 无知识命中警告。
- 建议动作条。

这些区块各自都在告诉坐席“现在该怎么处理”，容易造成重复、割裂和误判。P3-06U-12D 将它们合并为统一的“坐席判断区”。

## 2. 已完成改动

### 2.1 坐席判断区

新增统一判断区，展示：

| 信息 | 用途 |
|---|---|
| 坐席判断 | 当前整体处理模式，例如人工接管、填入草稿、补知识 |
| 客户 | 当前会话对象、主题和优先级 |
| AI 草稿 | 当前策略草稿或待审草稿，展示置信度与摘要 |
| 人审原因 | 当前为什么需要人工把关 |
| 知识依据 | 知识命中数量、引用数量、补知识入口 |
| 下一步 | 策略建议动作 |
| 安全门禁 | 明确当前为只读试算，不保存、不外发 |

### 2.2 策略试算接入

“试算当前会话”继续调用：

```text
POST /api/rpa-copilot/strategy-dry-run
```

结果直接进入坐席判断区，不再单独渲染一块“回复策略摘要”。

### 2.3 草稿带入回复框

新增“带入回复框”按钮：

- 只把策略草稿写入当前前端编辑框。
- 自动添加内部备注，说明草稿来自策略试算。
- 不保存数据库。
- 不进入待发送。
- 不触发 worker。
- 不外发到任何平台。

### 2.4 业务语言兜底

RPA 策略枚举和门禁原因不再直接暴露到界面。

已处理：

- `delivery_mode`
- `intent`
- `next_best_action`
- `guardrail.reasons`

未知枚举不回退显示原始英文值，而是显示保守中文兜底。

## 3. 保留边界

当前仍然是内部研究 dry-run：

- 未接真实微信、企微、抖音、小红书、淘宝、京东、拼多多。
- 未读取平台账号。
- 未做 RPA 观察真实页面。
- 未自动填入真实平台后台。
- 未点击发送。
- 未创建真实外发任务。
- 未对外承诺平台自动回复已接通。
